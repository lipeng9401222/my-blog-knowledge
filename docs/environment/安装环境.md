## 安装步骤

[<a href="#安装清单">清单</a>](?file=012-快速入门/001-安装环境#安装清单)中的工具并非所有场景都需要安装，一般来说可以划分为两大类：
- 如果你是一个售前、测试、或者交付，目标只是把程序运行起来，那么只需要安装[<a href="#生产环境安装">生产环境</a>](?file=012-快速入门/001-安装环境#生产环境安装)即可；
- 如果你是一个开发，目标是在IDE中运行系统，那么需要参照[<a href="#开发环境安装">开发环境安装</a>](?file=012-快速入门/001-安装环境#开发环境安装)或者[<a href="#开发环境快速安装">开发环境快速安装</a>](?file=012-快速入门/001-安装环境#开发环境快速安装)章节进行处理。

### 生产环境安装

生产环境一般只需要安装`数据库、数据库连接工具、tomcat`3项内容即可，前后分离部署的项目，还需要额外安装`nginx`。

### 开发环境安装
后端开发环境需要安装`eclipse、jdk、maven、git、数据库、数据库连接工具`，前端开发环境需要安装`vscode、nvm、nodejs`，下面会给出相关安装指南，详细的可以访问前端发布的[VUE环境安装指南](http://192.168.219.170/docs/vue/latest/frame/getting-started/development-environment/#huan_jing_zhun_bei)。

### 后端开发环境快速安装

#### 配置nvm源
```bash
# 可通过以下命令查看镜像源
npm config get registry
```
- 由于使用官方镜像地址会比较慢，需要按以下方式切换镜像源

1、方式一：命令行操作
```bash
nvm node_mirror  https://npmmirror.com/mirrors/node/
nvm npm_mirror https://npmmirror.com/mirrors/npm/
```


2、方式二：修改配置文件(直接修改nvm安装目录的settings.txt)
```txt
root: E:\nvm
path: E:\nodejs
node_mirror: https://npmmirror.com/mirrors/node/
npm_mirror: https://npmmirror.com/mirrors/npm/
```

##### 配置npm源(非必须，可跳过)
- 开源环境下，直接按以下切到 tb源即可

```bash
# 查看npm配置列表
npm config list
# 查看npm镜像地址 默认npm镜像：https://registry.npmjs.org/
npm config get registry
# 更新到淘宝最新地址
npm config set registry https://registry.npmmirror.com
```

##### 配置公司私有源
由于需要用到公司自己的源，因此`npm`源也要能够切换
`nrm` 是一个 `npm` 源管理器，允许快速地在 `npm`源间切换

##### 安装nrm
```bash
# 全局安装nrm- 如果上面已经切换过，直接运行这个
npm install -g nrm

# 如果之前未提前切换到tb源，可能会卡住，可以添加registry参数，临时切换源进行安装
npm install -g nrm  --registry=https://registry.npmmirror.com
```

##### 添加并切换公司源
```bash
# 设置新点源 通过 nrm add [名称] [地址]
nrm add epoint http://192.168.0.99:8081/nexus/repository/npmpublic/
# 切换到新点源
nrm use epoint
```
接下来需要登录到这个源，<font style="color:red">只需要进行一次，账号就是 maven的settings.xml里的 epoint-nexus 下的</font>
```bash
# 登录 按照提示输入 用户名  账号  密码
npm login
```
登录完成后`npm` 会将登录凭据保存在 `~/.npmrc` 文件中
```bash
# 登录完成后 测试是否连上
nrm test
```

##### 安装pnpm(组件化vue开发必须)
```bash
# 全局安装pnpm 目前推荐使用 v10 版本
npm install -g pnpm@10
```

#### 如何配置nginx


- 按此路径的nginx配置如下：
 - <font style="color:red">http</font>节点
```nginx
    upstream epoint-web {
        # 指向到真实的后端地址
        server 127.0.0.1:8080;
    }
```
 - 添加一个<font style="color:red">server</font>节点：
```nginx
server{
    listen 8897;
    server_name localhost;
    # 指向实际的根目录，按实际修改
    root E:/dist;

    #后端接口路由，如果单个vue工程有多个后端，其他后端接口也要仿照这个建立一组location
    location /epoint-web/rest {
      #指向实际upstream名
      proxy_pass http://epoint-web;
      proxy_set_header    Host             $http_host;
      proxy_set_header   X-Real-IP         $remote_addr;
      proxy_set_header   X-Forwarded-For   $proxy_add_x_forwarded_for;
      proxy_set_header   X-Forwarded-Proto $scheme;
    }

    # 静态资源路由
    location /epoint-web {
      try_files $uri $uri/ /epoint-web/index.html;
    }

   }
```

- 按照上述配置，可通过`http://localhost:8897/epoint-web`访问系统

双击运行nginx.exe即可
