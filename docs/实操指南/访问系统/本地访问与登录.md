# 本地访问与登录

## 目的

在后端与前端都启动完成后，确认：

- 数据库表已自动初始化
- 页面可访问（后端直连 / 前端 dev server）
- 授权（注册码）与登录流程可走通

## 前置条件

- 后端已启动：[后端：Tomcat 插件（war）启动](../开发环境启动/后端-tomcat插件启动.md)
- 前端已启动（如你选择包含前端）：[前端：Vue 组件化启动](../开发环境启动/前端-vue组件化启动.md)

## 步骤 1：确认数据库表初始化

原始文档参考：

- [访问系统](file:///Users/juanjuan/Projects/personal/myknowledge/docs/environment/01-快速入门/访问系统.md#L16-L20)

在你的业务库执行：

```sql
show tables;
```

预期：能看到大量业务表（而不是只有你手工建的测试表）。

如果没有表：

- 回到后端启动日志，找数据库连接/初始化相关报错
- 优先确认 `jdbc.properties` 连接的是否就是你这个库

## 步骤 2：确认访问地址

### 方式 A：后端直连（必须能通）

```text
http://localhost:8080/epoint-web
```

原始文档参考：

- [访问系统：访问系统](file:///Users/juanjuan/Projects/personal/myknowledge/docs/environment/01-快速入门/访问系统.md#L21-L25)

### 方式 B：前端 dev server（如果你启动了前端）

用 `pnpm run dev` 输出的地址访问（例如 `http://localhost:5173`），并确认页面请求能正常代理到后端。

## 常见问题与排查

### 1) 页面能打开但一直转圈或接口报错

优先检查：

- 浏览器 Network 是否出现 401/403/500
- 后端控制台是否出现异常栈
- 前端代理是否正确指向 `http://localhost:8080`

### 2) 访问 404

优先检查：

- context path 是否是 `/epoint-web`
- 你访问的 URL 是否少了 `/epoint-web`
