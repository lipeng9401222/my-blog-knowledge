# uni-app 一键发布到应用市场完整操作指南

> 适用项目：uni-app + HBuilderX 构建的 Android App

---

## 一、什么是「一键发布」

HBuilderX 从 **3.6.7+** 版本起内置了 `uni 应用商店分发服务`（俗称"一键发布"）。它的本质是：

- HBuilderX 在本地完成打包后，**自动调用各应用市场的开放平台 API**
- 把同一个 APK 分发上传到 华为 / 小米 / OPPO / VIVO / 荣耀 / 应用宝 等多家市场
- 自动填写版本说明、上传素材、提交审核

**核心价值**：把"打包 → 登录 N 个市场后台 → 分别上传 APK → 分别填表 → 分别提审"的重复工作，压缩成**一次操作**。

> ⚠️ 一键发布只是**自动化提交流程**，每家市场仍由人工审核，审核周期不变（1~5 个工作日）。

## 二、前置条件检查清单

在使用一键发布之前，必须满足以下所有条件：

### 2.1 HBuilderX 版本

- [x] HBuilderX **3.6.7 及以上**版本（推荐使用最新正式版）
- [x] 菜单路径：HBuilderX → 关于 HBuilderX，确认版本号

### 2.2 各市场开发者账号已就绪

| 市场 | 开发者后台 | 必须完成的事项 |
|------|-----------|---------------|
| 华为应用市场 | https://developer.huawei.com/ | 企业实名认证 + 应用已创建（包名 `com.yourcompany.app`） |
| 荣耀应用市场 | https://developer.honor.com/ | 企业实名认证 + 应用已创建 |
| 小米应用商店 | https://dev.mi.com/ | 企业实名认证 + 应用已创建 |
| OPPO 软件商店 | https://open.oppomobile.com/ | 企业实名认证 + 应用已创建 |
| VIVO 应用商店 | https://dev.vivo.com.cn/ | 企业实名认证 + 应用已创建（按需） |
| 应用宝 | https://open.tencent.com/ | 企业实名认证 + 应用已创建（按需） |

> 必须先在每个市场**手动创建至少一次应用**（首次提审），获得"应用 ID / 包标识"，一键发布才能识别后续的更新版本。

### 2.3 每家市场的 API 密钥已申请

一键发布依赖各市场的开放平台 API，需要**逐个**到对应后台申请：

| 市场 | 需要的密钥 | 申请位置 |
|------|-----------|---------|
| 华为 | `Client ID` + `Client Secret` | AppGallery Connect → 我的项目 → 项目设置 → API 密钥 |
| 荣耀 | `Client ID` + `Client Secret` | 荣耀开发者中心 → 应用 → API 服务 → API 密钥 |
| 小米 | `AccessKey` + `SecretKey` | 小米开发者后台 → 账号管理 → API 接入 |
| OPPO | `Client ID` + `Client Secret` | OPPO 开放平台 → 应用 → API 接入 |
| VIVO | `AccessKey` + `SecretKey` | VIVO 开发者中心 → 应用 → API 配置 |
| 应用宝 | 邮箱 + 密码 + 应用 KEY | 应用宝开放平台 → 应用 → API 配置 |

### 2.4 项目签名与备案

- [x] 签名证书：`your_app.keystore`（别名 `your_alias`）
- [x] 证书 MD5：[你的证书MD5指纹]
- [x] ICP 备案号：[你的ICP备案号]
- [x] 包名固定：`com.yourcompany.app`

> **极其重要**：每家市场首次创建应用时上传的证书 MD5 必须与 `your_app.keystore` 一致；后续一键发布更新版本时如果 MD5 不一致会被直接驳回。

### 2.5 上架素材已准备

参考 [华为AppGallery上架指南](./华为AppGallery上架指南.md) 第 3.1 节的提审材料清单：

- [ ] 应用图标 216×216 PNG（圆角）
- [ ] 应用截图 ≥ 3 张（1080×1920 或 1080×2340）
- [ ] 应用简介（15-80 字）
- [ ] 应用详细描述（100-4000 字）
- [ ] 隐私政策链接：`https://your-domain.com/privacy.html`
- [ ] 用户协议链接：`https://your-domain.com/user-agreement.html`
- [ ] 测试账号

## 三、完整操作流程

### 步骤 1：在各市场后台创建应用并完成首次提审

> ⚠️ **一键发布无法替代首次创建应用**。每家市场都必须先**手动创建应用 + 走完一次完整提审**，之后的版本迭代才能用一键发布。

参考 [华为AppGallery上架指南](./华为AppGallery上架指南.md) 完成华为首次上架，其他市场流程类似（见 Huawei 指南第十章）。

### 步骤 2：申请各市场 API 密钥

以华为为例：

1. 登录 [AppGallery Connect](https://developer.huawei.com/consumer/cn/service/josp/agc/index.html)
2. 选择"我的项目" → 选中项目 → 点击左下角"项目设置"
3. 切换到"API 管理"页签 → 启用"AppGallery Connect API"
4. 在"用户与权限" → "API 密钥"中创建一个 `Client ID` + `Client Secret`
5. **保存好这两个值**，HBuilderX 配置时需要

其他市场流程大同小异，统一在"开放平台/API 服务/账号管理"中找。

### 步骤 3：HBuilderX 配置应用商店账号

1. 打开 HBuilderX
2. 顶部菜单：**发行 → 原生 App-发布到各应用市场**
3. 第一次使用会要求登录 DCloud 账号（开发者账号）
4. 登录后进入"应用商店账号管理"页面
5. 依次点击各市场的"添加账号"按钮：
   - **华为应用商店**：填入 `Client ID` + `Client Secret`
   - **荣耀应用商店**：填入 `Client ID` + `Client Secret`
   - **小米应用商店**：填入 `AccessKey` + `SecretKey`
   - **OPPO 软件商店**：填入 `Client ID` + `Client Secret`
   - **VIVO 应用商店**（如需）：填入 `AccessKey` + `SecretKey`
   - **应用宝**（如需）：填入登录邮箱 + 密码 + 应用 KEY
6. 每个市场点击"测试连接"，确保认证通过
7. HBuilderX 会把这些密钥**加密保存在本地**

### 步骤 4：填写应用基础信息（一次配置多次复用）

在"一键发布"主界面：

1. **应用名称**：[你的应用名称]
2. **应用包名**：`com.yourcompany.app`
3. **应用分类**：教育 → 考试学习
4. **应用简介**：（15-80 字）

   ```
   [应用名称]——一句话简介，描述应用核心价值。
   ```

5. **应用详细描述**：（参考 [华为AppGallery上架指南 第 3.3 节](./华为AppGallery上架指南.md)）
6. **应用图标**：上传 `static/images/logo_round.png`（216×216 圆角 PNG）
7. **应用截图**：上传 5 张 1080×1920 截图
8. **隐私政策**：`https://your-domain.com/privacy.html`
9. **用户协议**：`https://your-domain.com/user-agreement.html`
10. **联系邮箱 / 客服电话**：填入企业联系方式

> 这些信息会被 **同时同步到所有勾选的应用市场**，无需重复填写。
> 部分市场对字数、图片尺寸有差异要求，HBuilderX 会高亮提示不合规项。

### 步骤 5：选择目标市场 + 配置打包

1. 在"目标市场"区域勾选要发布的市场：
   - ☑ 华为应用商店
   - ☑ 荣耀应用商店
   - ☑ 小米应用商店
   - ☑ OPPO 软件商店
   - ☐ VIVO（按需）
   - ☐ 应用宝（按需）

2. **打包配置**：
   - 包名：`com.yourcompany.app`
   - 证书：选择 `your_app.keystore`
   - 证书别名：`your_alias`
   - 证书库密码 / 证书私钥密码：见 [华为AppGallery上架指南 第 2.1 节](./华为AppGallery上架指南.md)
   - targetSdkVersion：`34`
   - minSdkVersion：`21`

3. **防重签**（强烈建议）：
   - ☑ 包名
   - ☑ 证书

4. **打包模式**：
   - ⦿ 传统打包（搭配证书防重签时**必选**）

> ⚠️ 一键发布上传给各市场的是**正式签名 APK**，必须打正式包，不能打调试基座。

### 步骤 6：填写版本信息

1. **versionName**：例如 `1.0.0`（与 manifest.json 一致）
2. **versionCode**：例如 `100`
3. **更新说明**（每家市场都会用这段文案）：

   ```
   [应用名称] 1.0.0 首发版本：
   1. AI 智能对话（小鹿博士），支持语音和文字交互
   2. 智能志愿推荐："冲、稳、保"三档方案
   3. 院校信息查询：覆盖 2800+ 所高校
   4. 专业百科：覆盖 500+ 本专科专业
   5. 多元升学测评：MBTI / 霍兰德职业兴趣等
   ```

4. **审核测试账号**：填入手机号 + 验证码获取方式
5. **审核备注**：

   ```
   测试账号：13800000000（动态短信验证码登录）
   核心功能路径：首页 → AI 对话 / 院校查询 / 智能填报
   ```

### 步骤 7：执行一键发布

1. 检查所有必填项无红色提示
2. 点击 **「开始发布」** 按钮
3. HBuilderX 会按顺序执行：
   1. 本地打包正式签名 APK
   2. 调用各市场 API 创建新版本记录
   3. 上传 APK 到各市场
   4. 自动同步素材（图标、截图、描述）
   5. 触发提审
4. 整个过程通常需要 **5-15 分钟**（取决于网速和市场响应）
5. 实时进度会在 HBuilderX 的"控制台"输出：

   ```
   [华为] 正在打包 ......... 完成
   [华为] 正在上传 APK ..... 完成
   [华为] 正在更新素材 ..... 完成
   [华为] 已提交审核 ✓
   [小米] 正在上传 APK ..... 完成
   [小米] 已提交审核 ✓
   [OPPO] 正在上传 APK ..... 完成
   [OPPO] 已提交审核 ✓
   [荣耀] 正在上传 APK ..... 完成
   [荣耀] 已提交审核 ✓
   ```

### 步骤 8：审核状态跟踪

发布完成后，可以在 HBuilderX → 发行 → 一键发布 → "**发布历史**"页签查看：

| 市场 | 版本 | 提交时间 | 状态 |
|------|------|---------|------|
| 华为 | 1.0.0 | 2026-05-28 | 审核中 |
| 小米 | 1.0.0 | 2026-05-28 | 已通过 |
| OPPO | 1.0.0 | 2026-05-28 | 审核中 |
| 荣耀 | 1.0.0 | 2026-05-28 | 已驳回 ⚠️ |

- 审核中：等待人工审核（1-5 天）
- 已通过：可在该市场搜索到应用
- 已驳回：点击查看驳回原因，修改后再次一键发布

## 四、版本迭代流程（最佳实践）

首次上架完成后，后续版本更新流程极简：

```
1. 修改代码 → 测试通过
2. 修改 manifest.json：
   - versionName：1.0.0 → 1.0.1
   - versionCode：100  → 101
3. HBuilderX → 发行 → 一键发布
4. 选择目标市场（默认上次的勾选）
5. 填写本次更新说明
6. 点击「开始发布」
7. 等待自动提审完成
```

整个过程通常 < 10 分钟，相比手动到 4 个市场分别上传**省去 80% 时间**。

## 五、各市场常见差异与适配

### 5.1 截图尺寸差异

| 市场 | 推荐尺寸 | 数量要求 |
|------|---------|---------|
| 华为 | 1080×1920 / 1080×2340 | ≥ 3 张 |
| 荣耀 | 1080×1920 / 1080×2340 | ≥ 3 张 |
| 小米 | 1080×1920 | ≥ 3 张，≤ 8 张 |
| OPPO | 1080×1920 / 1080×2340 | ≥ 3 张 |
| VIVO | 1080×1920 | ≥ 3 张 |

**建议**：统一准备 5 张 **1080×1920** 截图，所有市场通用。

### 5.2 应用描述字数限制

| 市场 | 简介字数 | 详细描述字数 |
|------|---------|-------------|
| 华为 | 15-80 | 100-4000 |
| 荣耀 | 15-80 | 100-4000 |
| 小米 | 不限 | 100-2000 |
| OPPO | 不限 | 100-2000 |

**建议**：把详细描述控制在 **100-2000 字**之内，所有市场通用。

### 5.3 隐私合规审查严格度

由严到松：**华为 > 小米 > OPPO > 荣耀**

通用要求（已实现）：
- 首次启动隐私弹窗（`androidPrivacy.json`）
- 同意前不发起任何网络请求
- 权限动态申请（`permission.js`）
- 隐私政策中列出所有第三方 SDK

### 5.4 应用类目映射

不同市场的"教育"分类路径略有不同，HBuilderX 会自动映射：

| 市场 | 推荐分类路径 |
|------|-------------|
| 华为 | 教育 → 考试学习 |
| 荣耀 | 教育 → 学习辅导 |
| 小米 | 教育学习 → 考试服务 |
| OPPO | 教育 → 升学考试 |

## 六、常见问题与排查

### 6.1 「测试连接」失败

**可能原因**：
- API 密钥填错（Client ID / Secret 顺序颠倒）
- 该市场的 API 服务未启用（需在开发者后台手动开启）
- 网络代理拦截（关闭 VPN 或科学上网工具）
- 密钥已过期（部分市场密钥有效期 6-12 个月）

**排查**：
1. 直接复制密钥到 [Postman / curl](https://example.com) 测试 API
2. 在该市场开发者后台重新生成密钥

### 6.2 上传 APK 时报「证书不匹配」

**原因**：本次打包的 APK 证书 MD5 与该市场首次上传的 APK 不一致。

**解决**：
- 确认使用的是同一个 `your_app.keystore`
- 校验证书 MD5：[你的证书MD5指纹]
- 如果误用了其他证书，**只能通过该市场客服申请重置签名**（华为/小米支持，部分市场需重新创建应用）

### 6.3 versionCode 必须递增

**原因**：每家市场都要求 `versionCode` **严格递增**，不能与已上传的版本相同或更小。

**解决**：每次发布前手动递增 `versionCode`，建议规则：
- 主版本：100、200、300
- 修订版本：101、102、103…

### 6.4 部分市场审核失败，部分成功

**正常现象**。一键发布只是**并行提交**，各家市场独立审核。失败的市场点击查看具体原因后，**只针对失败市场**重新一键发布即可（其他成功市场无需重复操作）。

### 6.5 一键发布对图片的额外要求

- 图标必须是 PNG 格式
- 华为要求**圆角**，小米/OPPO/VIVO 要求**无圆角正方形**
- HBuilderX 会自动适配大部分情况，但**建议同时准备两版图标**：
  - `logo.png` 无圆角正方形 → 小米/OPPO/VIVO/应用宝
  - `logo_round.png` 圆角 → 华为/荣耀

> 参考 `static/images/` 目录下的图标文件。

### 6.6 一键发布上传中断

**可能原因**：网络抖动、HBuilderX 崩溃、电脑休眠

**解决**：HBuilderX 支持**断点续传**，重新执行一键发布即可。已成功上传的市场会自动跳过。

## 七、一键发布 vs 手动上架对比

| 维度 | 一键发布 | 手动上架 |
|------|---------|---------|
| 首次上架 | ❌ 不支持 | ✅ 支持 |
| 版本迭代 | ✅ 5-10 分钟 | ❌ 60-90 分钟（4 个市场） |
| 素材同步 | ✅ 一次填写多端复用 | ❌ 每家分别上传 |
| 审核进度跟踪 | ✅ 集中查看 | ❌ 分别登录 4 个后台 |
| 驳回处理 | ✅ 仅重提失败市场 | ❌ 手动判断 |
| 配置成本 | ❌ 首次需配置 4-6 套 API 密钥（半天） | ✅ 无需配置 |
| 灵活度 | ❌ 无法做差异化文案 | ✅ 各市场独立填写 |

**结论**：
- **首次上架**：用手动方式（参考 [华为AppGallery上架指南](./华为AppGallery上架指南.md)）
- **后续迭代**：强烈推荐一键发布

## 八、本项目落地步骤建议

### 阶段 1：首次上架（约 7-10 天）
1. 按 [华为AppGallery上架指南](./华为AppGallery上架指南.md) 完成华为首次上架
2. 同步在小米/OPPO/荣耀分别手动创建应用并提审（参考 Huawei 指南第十章）

1. 在 4 个市场后台逐个申请 API 密钥
2. HBuilderX → 发行 → 一键发布 → 配置 4 套账号
3. 测试连接通过

### 阶段 3：版本迭代（持续）
1. 每次发版只需：改 versionCode → 一键发布 → 等待审核
2. 后续完全告别"手动登录 4 个后台 + 重复上传"

## 十、注意事项汇总

1. **首次必须手动**：一键发布是为版本迭代设计的，首次上架仍需手动完成 + 通过审核
2. **签名永远一致**：所有市场所有版本必须用同一个 `your_app.keystore`
3. **versionCode 严格递增**：每发一次必须 +1
4. **API 密钥保密**：不要把密钥提交到 Git（已在 `.gitignore` 中排除签名相关文件，密钥仅本地保存）
5. **审核期间稳定后端**：一键发布提审后，确保后端 API、隐私政策链接、用户协议链接都可正常访问
6. **测试账号长期有效**：审核测试账号要保证审核期间能正常登录使用
7. **谨慎一键回滚**：一键发布的版本无法批量回滚，需到各市场后台分别下架
