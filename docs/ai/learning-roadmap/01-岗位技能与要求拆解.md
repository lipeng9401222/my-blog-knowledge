# 01 岗位技能与要求拆解

## 岗位名称

`AI应用工程师（工业数据/工业AI方向）`

从截图内容看，这个岗位的关键关键词包括：

| 类别 | 关键词 |
|---|---|
| 大模型 | DeepSeek、Qwen、提示词工程、RAG、LangChain、LlamaIndex、API 调用 |
| 数据工程 | DCS、SCADA、时序数据、清洗、特征工程、标注、高质量数据集 |
| 机器学习 | 传统 ML、Scikit-learn、PyTorch、TensorFlow、LSTM、Transformer |
| 后端集成 | FastAPI、Flask、Django、RESTful API、Docker |
| 工业系统 | MES、工业互联网平台、工艺优化、预测性维护、质量检测、智能排产 |
| 协作交付 | 与工艺/运维团队协作、收集反馈、优化模型、技术文档、汇报材料 |

## 岗位职责逐条翻译

### 职责 1：AI 应用开发与部署

原始要求：

> 基于大模型（DeepSeek/Qwen 等）及传统 ML 框架，开发工艺优化、预测性维护、质量检测、智能排产等应用。

实际含义：

| 要求 | 你需要会什么 |
|---|---|
| 基于大模型开发 | 会调用大模型 API，会写提示词，会做 RAG，会处理模型输出 |
| 基于传统 ML 框架开发 | 会用 scikit-learn 做分类、回归、异常检测，会做指标评估 |
| 工艺优化 | 能理解流程数据和工艺参数之间的关系，至少能做数据分析和建议 |
| 预测性维护 | 能基于温度、振动、电流、压力等时序数据预测故障风险 |
| 质量检测 | 能做缺陷分类、异常识别，若涉及图像要了解 CV 基础 |
| 智能排产 | 了解约束、规则、优化目标，初期可用规则和启发式方法实现 Demo |

优先学习：

| 优先级 | 学习内容 |
|---|---|
| P0 | Python、pandas、scikit-learn、FastAPI、大模型 API |
| P0 | RAG 知识库、工业文档问答、模型服务接口 |
| P1 | PyTorch、LSTM、Transformer、时间序列预测 |
| P2 | 优化算法、排产算法、工业 CV |

### 职责 2：工业数据工程

原始要求：

> 处理 DCS/SCADA 时序数据，完成清洗、特征工程、标注，构建高质量数据集。

实际含义：

| 要求 | 你需要会什么 |
|---|---|
| DCS/SCADA 时序数据 | 知道工业传感器数据通常是时间戳 + 点位 + 数值 + 状态 |
| 清洗 | 处理缺失值、重复值、异常值、单位差异、时间乱序 |
| 特征工程 | 计算滑窗均值、最大值、最小值、标准差、变化率、滞后特征 |
| 标注 | 根据故障记录、报警、工单、阈值规则给数据打标签 |
| 高质量数据集 | 能输出可复用的数据集、数据说明、字段说明、质量报告 |

这是岗位最基础也最容易被低估的能力。工业 AI 项目常见失败原因不是模型不够高级，而是数据不可用、标签不可靠、业务定义不清。

你至少要能独立完成：

1. 读取 CSV/Excel/数据库中的时序数据。
2. 按时间戳排序、重采样、补齐缺失点。
3. 找出异常值并解释处理策略。
4. 用滑动窗口构造模型输入特征。
5. 输出训练集、验证集、测试集。
6. 写一份数据质量分析报告。

### 职责 3：模型与服务集成

原始要求：

> 使用 FastAPI/Flask 封装模型为 API，对接 MES/工业互联网平台，保障稳定性与实时性。

实际含义：

| 要求 | 你需要会什么 |
|---|---|
| FastAPI/Flask | 能写 RESTful API，接收 JSON，返回预测结果 |
| 模型封装 | 能加载模型文件，封装预测函数，处理输入校验 |
| 对接 MES/平台 | 知道业务系统通过 HTTP、消息队列、数据库、OPC UA 等方式传数据 |
| 稳定性 | 会处理异常、日志、超时、健康检查、错误码 |
| 实时性 | 知道批处理和实时推理的区别，能控制接口耗时 |

最低可交付标准：

| 接口 | 作用 |
|---|---|
| `GET /health` | 服务健康检查 |
| `POST /predict` | 输入传感器数据，返回风险分数或预测值 |
| `POST /rag/query` | 输入问题，返回知识库答案和引用来源 |
| `GET /version` | 返回模型版本和服务版本 |

### 职责 4：私域知识库构建

原始要求：

> 基于 RAG 构建工业智能知识库，完成规程、手册、案例等文档的语料清洗、向量化与集成。

实际含义：

| 要求 | 你需要会什么 |
|---|---|
| 语料清洗 | 处理 PDF、Word、Markdown、表格，去页眉页脚和无效内容 |
| 文档切分 | 按标题、段落、长度、语义边界切 chunk |
| 向量化 | 用 embedding 模型把文本变成向量 |
| 检索 | 根据问题召回相关文档片段 |
| 生成 | 把检索结果交给大模型回答 |
| 引用来源 | 告诉用户答案来自哪份文档、哪一段 |

岗位需要的不是“会聊天机器人”，而是能把企业文档变成可检索、可追溯、可维护的知识系统。

RAG 项目必须特别注意：

1. 不允许模型胡编规程。
2. 答案要带来源。
3. 检索不到时要明确说不知道。
4. 文档更新后要能重新入库。
5. 要能处理术语、设备编号、工艺缩写。

### 职责 5：跨部门协同

原始要求：

> 与工艺/运维团队协作，收集反馈优化模型；编写技术文档及汇报材料。

实际含义：

| 要求 | 你需要会什么 |
|---|---|
| 理解非技术需求 | 把“设备不稳定”“产线波动大”翻译成数据问题 |
| 收集反馈 | 记录误报、漏报、不可解释结果和业务限制 |
| 优化模型 | 根据反馈调整特征、阈值、提示词、检索策略 |
| 技术文档 | 写接口文档、数据说明、模型说明、部署说明 |
| 汇报材料 | 用业务语言说明收益，而不是只说模型指标 |

## 任职要求拆解

### 基础条件

| 原始要求 | 解读 | 从零准备建议 |
|---|---|---|
| 本科及以上，计算机、AI、自动化、软件、数学、统计等相关专业 | 偏技术背景，自动化和工业背景也认可 | 如果专业不完全匹配，用项目作品补 |
| 2 年以上 AI/ML 开发经验，优秀硕士应届生可放宽 | 期望不是纯小白，至少能独立开发 | 用 4 个完整项目模拟 1 年左右项目经验 |

如果没有 2 年经验，简历必须补足：

1. 有可运行代码。
2. 有完整项目文档。
3. 有数据处理和模型评估记录。
4. 能解释为什么这样设计。
5. 能说明工业业务价值。

### 技术能力

| 原始要求 | 岗位重要度 | 学到什么程度 |
|---|---:|---|
| 熟练 Python | P0 | 能写模块化脚本、处理文件、调用 API、读写数据库 |
| 熟练 pandas/NumPy | P0 | 能独立清洗、聚合、重采样、特征工程 |
| 熟练 scikit-learn | P0 | 能完成分类、回归、异常检测、模型评估 |
| 熟悉 PyTorch 或 TensorFlow | P1 | 能训练简单神经网络，理解 Dataset、DataLoader、训练循环 |
| 熟悉大模型应用 | P0 | 会 API 调用、提示词、RAG、工具调用基本思路 |
| 熟悉 LangChain/LlamaIndex | P0 | 能实现文档加载、切分、向量检索、问答链 |
| 熟悉 RESTful API | P0 | 能设计接口、处理请求、返回错误码 |
| 熟悉 FastAPI/Flask/Django | P0 | 优先 FastAPI，其次 Flask，Django 可了解 |
| 熟悉 Docker 基本操作 | P0 | 能写 Dockerfile、构建镜像、运行服务 |

### 加分项

| 加分项 | 为什么加分 | 学习策略 |
|---|---|---|
| 流程行业项目经验 | 工业 AI 落地依赖场景理解 | 用公开数据集模拟化工、电力、设备维护场景 |
| 工业时序预测 | 直接对应预测性维护和质量预测 | 做一个 LSTM/Transformer 对比项目 |
| Dify/n8n/Agent 框架 | 企业可能想快速搭应用 | 了解工作流编排，做一个简单自动化 Demo |
| MES/SCADA/DCS 数据交互 | 直接影响系统集成 | 先理解概念，再模拟 API/MQTT/时序库对接 |
| 工艺原理或安全规范 | 能和业务团队沟通 | 选一个行业做轻量背景学习 |

| 能力域 | 重要级别 | 目标水平 | 验收方式 |
|---|---:|---|---|

| 机器学习与时序预测 | P0 | 能完成 baseline、训练、评估、解释 | 完成预测性维护模型 |

## 不建议一开始深学的内容

为了避免学习路线失控，以下内容前 3 个月不建议深挖：

| 内容 | 原因 | 什么时候学 |
|---|---|---|
| 大模型预训练 | 与岗位应用落地关系不大 | 9 个月后，如有研究兴趣再学 |
| 复杂强化学习 | 岗位没有明确要求 | 智能排产深入阶段再学 |
| 视觉检测高阶算法 | 岗位提到质量检测，但截图没有明确 CV 栈 | 完成主线后作为加分项 |
| Kubernetes | 岗位只要求 Docker 基本操作 | API 服务熟练后再补 |
| 高阶数学推导 | 容易拖慢落地 | 跟着模型学习逐步补 |

## 这份岗位的真实难点

1. 技术跨度大：Python、ML、LLM、API、Docker、工业系统都要懂。
2. 数据质量难：工业数据经常缺失、漂移、噪声大、标签少。
3. 业务沟通难：工艺和运维团队表达问题的方式不一定是技术语言。
4. 落地要求高：Demo 会跑不够，还要稳定、可解释、可维护。
5. 经验门槛明显：JD 写了 2 年 AI/ML 经验，需要用项目作品弥补。

## 学习路线优先级建议

如果时间有限，按下面顺序学：

1. Python + pandas + NumPy。
2. 工业时序数据清洗与特征工程。
3. scikit-learn 分类、回归、异常检测。
4. FastAPI 封装预测接口。
5. 大模型 API + RAG 知识库。
6. Docker 部署。
7. PyTorch 时序预测。
8. MES/SCADA/DCS 概念与模拟对接。
9. 简历项目包装和面试表达。
