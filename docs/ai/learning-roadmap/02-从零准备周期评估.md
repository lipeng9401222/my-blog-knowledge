# 02 从零准备周期评估

## 评估前提

本评估基于以下前提：

| 项目 | 默认值 |
|---|---|
| 当前基础 | 会一点编程 |
| AI/ML 基础 | 默认没有系统经验 |
| 每天投入 | 2 小时 |
| 每周投入 | 6 天，约 12 小时 |
| 学习目标 | 入职能干活 |
| 岗位方向 | 工业数据方向 AI 应用工程师 |

这不是“看完课程”的周期，而是“能做出项目、能解释方案、能投递和面试”的周期。

## 总体结论

| 周期 | 现实状态 | 是否建议投递 |
|---|---|---|
| 1 到 2 个月 | 能写 Python 脚本，能做简单数据清洗 | 不建议正式投递，可投实习或练手 |
| 3 个月 | 能做传统 ML 小项目，理解工业时序基础 | 可尝试投初级 AI 应用/数据分析相关岗位 |
| 6 个月 | 有 2 到 3 个可展示项目，能做 RAG 和 API Demo | 可正式投递，重点投初级/转岗/项目型岗位 |
| 8 到 9 个月 | 有完整工业 AI 应用作品集，能讲清工程闭环 | 更接近 JD 中“入职能干活”的状态 |
| 12 个月 | 如果持续做真实项目，可接近 1 年项目经验的表达水平 | 可冲更高匹配度岗位 |

## 为什么不是 3 个月速成

这个岗位同时要求：

1. Python 工程能力。
2. 数据清洗和特征工程。
3. 传统机器学习。
4. 深度学习或时序预测。
5. 大模型应用和 RAG。
6. FastAPI/Flask 服务封装。
7. Docker 部署。
8. 工业系统和业务理解。
9. 文档、汇报、跨部门沟通。

每天 2 小时的情况下，3 个月约 144 小时。这个时间足够建立基础和做一个小项目，但不足以稳定覆盖岗位全部要求。更稳妥的准备周期是 6 到 9 个月。

## 3 个月目标

### 能达到什么

3 个月后，你应达到：

| 能力 | 目标状态 |
|---|---|
| Python | 能写脚本处理文件、函数、类、异常、日志 |
| pandas/NumPy | 能处理 CSV、Excel、缺失值、异常值、聚合统计 |
| scikit-learn | 能做分类、回归、异常检测 baseline |
| 工业数据 | 理解时间戳、传感器点位、报警、设备状态 |
| 项目产出 | 至少 1 个工业数据清洗项目和 1 个 ML 小项目 |

### 不应期待什么

3 个月后通常还不稳定：

1. 独立设计复杂 RAG 系统。
2. 熟练训练 LSTM/Transformer。
3. 独立对接 MES/SCADA/DCS。
4. 独立负责生产环境模型服务。
5. 面对真实工业数据能快速定位所有问题。

### 3 个月可投递方向

| 岗位 | 匹配程度 |
|---|---|
| AI 应用实习生 | 高 |
| 数据分析/数据处理实习 | 高 |
| Python 开发实习 | 中 |
| 初级算法工程师 | 中低 |
| 工业 AI 应用工程师正式岗 | 低到中，取决于项目作品 |

## 6 个月目标

### 能达到什么

6 个月后，你应达到：

| 能力 | 目标状态 |
|---|---|
| 数据工程 | 能完成较完整的工业时序数据清洗、特征工程、数据集构建 |
| 传统 ML | 能做模型选择、训练、评估、调参和误差分析 |
| 时序预测 | 能做 baseline 和简单 LSTM 模型 |
| 大模型应用 | 能调用 DeepSeek/Qwen API，完成提示词和 RAG Demo |
| API 服务 | 能用 FastAPI 封装预测接口和知识库问答接口 |
| Docker | 能把服务打包成本地镜像运行 |
| 文档 | 能写项目说明、接口说明、模型报告 |

### 6 个月作品要求

至少准备 3 个项目：

| 项目 | 要求 |
|---|---|
| 工业时序数据清洗管道 | 有代码、数据质量报告、特征工程说明 |
| 预测性维护模型 | 有 baseline、评估指标、模型解释 |
| 工业规程 RAG 知识库 | 有文档入库、检索、问答、引用来源 |

### 6 个月可投递方向

| 岗位 | 匹配程度 |
|---|---|
| AI 应用工程师初级岗 | 中到高 |
| 大模型应用工程师初级岗 | 中 |
| Python 后端 + AI 工具开发 | 中到高 |
| 数据开发/数据分析 + AI 方向 | 中到高 |
| 工业 AI 应用工程师 | 中，项目表达很关键 |

## 9 个月目标

### 能达到什么

9 个月后，你应达到：

| 能力 | 目标状态 |
|---|---|
| 工业数据闭环 | 能从数据采集、清洗、建模、接口、部署讲完整流程 |
| 时序建模 | 能比较传统 ML、统计方法、LSTM/Transformer 的优缺点 |
| RAG 工程 | 能解释切分、向量化、召回、重排、引用、幻觉控制 |
| 服务部署 | 能实现 FastAPI + Docker 的基础生产化结构 |
| 工业系统 | 能说明 MES、SCADA、DCS 的差异和数据交互方式 |
| 业务沟通 | 能把模型指标翻译成误报、漏报、停机风险、运维收益 |

### 9 个月作品要求

至少准备 4 个完整作品：

| 项目 | 要求 |
|---|---|
| 工业数据清洗与特征平台 | 有可复用模块和报告 |
| 预测性维护模型 | 有传统 ML + 深度学习对比 |
| 工业知识库 RAG | 有引用来源、评测集、失败案例分析 |
| 模型服务集成 Demo | 有 API、Docker、接口文档、模拟业务调用 |

### 9 个月可投递方向

| 岗位 | 匹配程度 |
|---|---|
| AI 应用工程师 | 高 |
| 工业 AI 应用工程师 | 中到高 |
| 大模型应用工程师 | 中到高 |
| Python AI 工程师 | 高 |
| 初级算法工程师 | 中 |

## 准备周期详细估算

| 模块 | 最低小时 | 稳妥小时 | 说明 |
|---|---:|---:|---|
| Python 基础与工程习惯 | 40 | 70 | 会一点编程可压缩，但仍要补工程化 |
| pandas/NumPy 数据处理 | 50 | 90 | 岗位最核心基础之一 |
| 机器学习与 scikit-learn | 60 | 100 | 分类、回归、异常检测、评估 |
| 深度学习与时序预测 | 60 | 120 | PyTorch、LSTM、Transformer 入门 |
| 大模型 API 与提示词 | 30 | 50 | 上手快，但要工程化 |
| RAG 知识库 | 60 | 100 | 文档处理、向量库、评估都要练 |
| FastAPI/Flask API | 40 | 70 | 必须能封装服务 |
| Docker 与部署 | 25 | 50 | 至少本地容器化 |
| 工业系统与场景 | 40 | 80 | MES/SCADA/DCS、流程行业概念 |
| 项目整合与面试 | 60 | 100 | 决定能不能转化成 offer |

按每周 12 小时计算：

| 总小时 | 约需周期 |
|---:|---:|
| 425 小时 | 35 到 36 周 |
| 600 小时 | 50 周左右 |

因此 36 周路线属于“聚焦岗位核心、压缩非必要内容”的可执行版本。

## 每月里程碑

| 月份 | 目标 | 可展示产出 |
|---|---|---|
| 第 1 月 | Python 和环境基础 | 文件处理脚本、数据读取脚本 |
| 第 2 月 | 数据清洗和特征工程 | 工业时序清洗 Notebook |
| 第 3 月 | 传统 ML 和异常检测 | 预测性维护 baseline |
| 第 4 月 | 深度学习和时序预测 | LSTM/Transformer 入门实验 |
| 第 5 月 | 大模型 API 和 RAG | 工业规程知识库 Demo |
| 第 6 月 | FastAPI 和 Docker | 模型服务接口 |
| 第 7 月 | 工业系统集成 | 模拟 MES/SCADA 数据流 |
| 第 8 月 | 综合项目 | 工业 AI 应用平台 Demo |
| 第 9 月 | 简历和面试 | 项目文档、面试题、技术汇报 |

## 判断自己能否投递的标准

满足以下 8 条中的 6 条，可以开始正式投递：

1. 能独立用 pandas 清洗一份时序数据。
2. 能解释缺失值、异常值、重采样、滑窗特征怎么处理。
3. 能用 scikit-learn 做一个分类或回归模型并解释指标。
4. 能调用大模型 API 完成一个业务问答任务。
5. 能实现一个带引用来源的 RAG Demo。
6. 能用 FastAPI 封装一个 `/predict` 接口。
7. 能用 Docker 跑起自己的服务。
8. 能用业务语言讲清楚一个工业 AI 项目的价值。

## 风险和应对

| 风险 | 表现 | 应对 |
|---|---|---|
| 只看课不做项目 | 知识点懂，但不会独立实现 | 每周必须提交代码或报告 |
| 数学卡住 | 花大量时间推公式 | 先会用模型，再补原理 |
| 大模型学得太散 | 工具很多，每个都浅尝 | 主线固定为 API + RAG + FastAPI |
| 工业概念太陌生 | 看不懂 DCS/SCADA/MES | 先理解数据流，不追求工艺专家水平 |
| 项目不可展示 | 代码零散、无文档 | 每个项目都写 README、接口说明、结果分析 |

## 最务实的结论

如果你每天坚持 2 小时：

1. 第 1 到 3 个月，不要急着说自己能胜任岗位，先把 Python 数据处理和 ML 项目打牢。
2. 第 4 到 6 个月，开始做 RAG、FastAPI、Docker，把能力从“会分析”变成“能交付应用”。
3. 第 7 到 9 个月，把工业场景、系统对接、项目文档和面试表达补齐，这是接近岗位要求的关键阶段。
