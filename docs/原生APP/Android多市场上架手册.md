# Android 多应用市场上架手册（OPPO / 小米 / VIVO / 荣耀 / 一加 / iQOO 等）

> 适用项目：uni-app + HBuilderX 云打包的 Android App
> 配套文档：[`华为AppGallery上架指南`](./华为AppGallery上架指南.md)（华为单独走 AppGallery 流程）
>
> 上架所需的签名、备案、隐私政策等关键信息全部沿用华为手册第二章内容，无需重复准备。

## 一、前期通用准备

### 1.1 项目核心资料（四市场+小市场全部复用）

| 项 | 值 | 备注 |
|------|------|------|
| 应用名称 | [你的应用名称] | 以软著名称为准 |
| 应用包名 | `com.yourcompany.app` | manifest.json `packagename` 字段 |
| 应用 appid | [你的appid] | DCloud 分配 |

| ABI | `arm64-v8a` | 已满足 64 位强制要求 |
| 主办单位 | [你的公司名称] | 企业开发者主体 |
| ICP 备案号 | [你的ICP备案号] | 主办单位：[你的公司名称] |
| 隐私政策 URL | https://your-domain.com/privacy.html | HTTPS 公网可访问 |
| 用户协议 URL | https://your-domain.com/user-agreement.html | HTTPS 公网可访问 |
| 应用图标 | 准备 216×216 圆角 PNG | 各市场可裁切复用 |
| 软件著作权 | 已具备 | 强烈建议提交 |

### 1.2 签名证书（所有市场共用同一份，**不可换证书**）

| 项 | 值 |
|------|-----|
| 证书文件 | `your_app.keystore`（项目根目录） |
| 证书别名 | `your_alias` |
| 证书库密码 | [你的证书库密码] |
| 证书私钥密码 | [你的证书私钥密码] |
| 签名算法 | RSA 2048 位 |
| 有效期 | 100 年 |
| MD5 指纹 | [你的证书MD5指纹] |

> ⚠️ **关键约束**：OPPO / 一加、VIVO / iQOO、荣耀、小米均做强签名校验，每次更新必须使用**同一证书**，否则直接拒审且无法升级覆盖安装。证书与密码遗失=应用作废，请离线多备份。

### 1.3 HBuilderX 云打包渠道策略（一次出多包）

> 详细打包步骤、防重签互斥规则等参见 [`华为AppGallery上架指南`](./华为AppGallery上架指南.md)。本节只列结论。

**渠道包勾选**：
- ☑ 无（通用包，给荣耀 / 一加 / iQOO / 三星 / 酷安 / 豌豆荚 / 魅族 / 联想等无独立渠道的市场）
- ☑ 华为应用商店（华为单独使用，详见华为手册）
- ☑ 小米应用商店
- ☑ OPPO
- ☑ VIVO
- ☑ 应用宝（腾讯）
- ☑ 360应用市场
- ☑ 百度（如需要）

**输出文件**（HBuilderX 实际命名以"渠道名"后缀为准）：

```
[你的appid]__1.0.0_general.apk          → 荣耀 / 一加 / iQOO / 三星 / 酷安 / 豌豆荚 / 魅族 / 联想等
[你的appid]__1.0.0_华为应用商店.apk      → 华为
[你的appid]__1.0.0_小米应用商店.apk      → 小米
[你的appid]__1.0.0_OPPO.apk             → OPPO
[你的appid]__1.0.0_VIVO.apk             → VIVO
[你的appid]__1.0.0_应用宝.apk           → 应用宝
[你的appid]__1.0.0_360应用市场.apk      → 360
[你的appid]__1.0.0_百度.apk             → 百度
```

**打包要点**：
- 模式：⦿ 打正式包；
- 防重签：☑ 包名（必勾）+ ☑ 证书（推荐，需配合"传统打包"，互斥"快速安心打包"）；
- ☑ 生成 SourceMap（便于错误堆栈还原）；
- ☐ 全部广告联盟 SDK（本项目无广告变现，全部不勾）。

## 二、四大主流市场详细上架流程

### 2.1 OPPO 软件商店

#### 2.1.1 基本信息

| 项 | 值 |
|------|-----|
| 开发者后台 | https://open.oppomobile.com/ |
| 上传 APK | `[你的appid]__1.0.0_OPPO.apk` |
| 审核周期 | 1-3 个工作日 |
| 账号类型 | 企业开发者 |
| 适配系统 | ColorOS（基于 Android） |

#### 2.1.2 注册开发者账号

1. 访问 https://open.oppomobile.com/ → 右上角"注册"。
2. 选择"企业开发者"，准备：
   - 营业执照副本扫描件；
   - 法人身份证正反面；
   - 企业对公账户信息；
   - 加盖公章的"OPPO 开发者授权书"（后台可下载模板）；
   - 企业邮箱。
3. 提交资质 → OPPO 人工审核（1-2 个工作日）→ 通过后进入"分发服务-应用与游戏"。

#### 2.1.3 创建应用并上传

1. 登录后台 → 左侧菜单"分发服务" → "应用与游戏" → "新增应用"。
2. 选择"应用" → 平台选"Android" → 上传 `_OPPO.apk` 包，系统自动解析包名 `com.yourcompany.app`、版本号、签名信息。
3. 填写基础信息：
   - 应用名称：**[你的应用名称]**（与软著一致，禁止"最/第一/国家级/官方"等绝对化用语）；
   - 应用副标题：AI 高考志愿填报助手；
   - 应用分类：教育 → 学习教育；
   - 应用语言：简体中文；
   - 关键词：志愿填报、高考、选大学、AI、升学规划（≤5 个）；
4. 上传图形素材：
   - 应用图标 216×216 圆角 PNG；
   - 应用截图 ≥4 张（1080×1920 或 1080×2340，竖屏真实截图）；
   - 应用宣传图 1 张（横版 1280×720，可选）。
5. 应用简介与详细描述：直接复用华为手册 §3.3 的简介与详描文案。

#### 2.1.4 资质与合规

1. **软件著作权**：上传 `软著.pdf` 扫描件（必填）。
2. **ICP 备案**：填写 [你的ICP备案号]，主办单位：[你的公司名称]。
3. **隐私政策**：填 `https://your-domain.com/privacy.html`，必须明确列出**每个动态权限的使用场景**（OPPO 复审重点）。
4. **用户协议**：填 `https://your-domain.com/user-agreement.html`。
5. **测试账号**：提供一个手机号 + 验证码登录方式（建议是已有数据的账号，让审核能看完整功能）。
6. **应用截图水印**：不能含其他品牌手机外观，建议无状态栏或使用 OPPO 状态栏。

#### 2.1.5 OPPO 特殊审核点

| 检查点 | 落地要求 |
|--------|----------|
| 签名一致性 | 每次更新必须用同一 `your_app.keystore`，**OPPO 对此最严格** |
| 用语合规 | 文案禁止绝对化用语（"最强 AI"→ 改为"智能 AI"） |
| 权限场景 | `RECORD_AUDIO`、`READ/WRITE_STORAGE` 必须在隐私政策中详细描述使用场景 |
| 隐私弹窗 | 首启必须先弹隐私协议，同意前禁止任何网络请求/数据采集 |
| 注销账号 | 必须在 App 内提供注销入口（已实现：我的-设置-注销账号） |
| ColorOS 兼容 | 建议在 ColorOS 13/14 真机回归一次推送、权限弹窗、字体放大场景 |

#### 2.1.6 提交审核

1. 全部信息填完 → "保存草稿" → 检查无误后 "提交审核"。
2. 审核状态在"应用列表"中查看；驳回会以邮件 + 站内信通知，按驳回原因修改后重新提交。
3. 通过后选择"立即上架"或"定时上架"，约 1-2 小时全网生效。

### 2.2 小米应用商店

#### 2.2.1 基本信息

| 项 | 值 |
|------|-----|
| 开发者后台 | https://dev.mi.com/ |
| 上传 APK | `[你的appid]__1.0.0_小米应用商店.apk` |
| 审核周期 | 1-2 个工作日（**四市场最快**，建议华为之后立即提审） |
| 账号类型 | 企业开发者 |
| 适配系统 | MIUI / HyperOS |

#### 2.2.2 注册开发者账号

1. 访问 https://dev.mi.com/ → 用小米账号登录 → "成为开发者"。
2. 选"企业开发者"，准备：
   - 营业执照；
   - 法人身份证；
   - 企业对公账户（用于打款验证）；
   - 加盖公章的授权书（后台模板）。
3. 小米通过对公账户打入随机金额，回填正确金额完成验证 → 1-3 个工作日完成认证。

#### 2.2.3 创建应用并上传

1. 后台 → "管理控制台" → "应用" → "新增应用" → 选"App"类型。
2. 填写应用名称 `[你的应用名称]`、应用类型"应用"、一级分类"教育" → 二级"学习"。
3. 上传 `_小米应用商店.apk`，系统解析后**校验 64 位**：必须包含 `arm64-v8a`（项目已满足）。
4. 上传素材：
   - 图标 256×256 PNG（小米也支持 512×512 高清图标，**强烈建议两份都上传**）；
   - 横版宣传图 1 张（1080×608）；
   - 截图 ≥4 张（1080×1920 或 1080×2340）。
5. 应用描述：复用华为手册 §3.3 文案。

#### 2.2.4 资质与合规

1. **软件著作权**：上传扫描件，证书名与应用名需一致（不一致需补"简称证明"）。
2. **ICP 备案**：必填 `[你的ICP备案号]`。
3. **隐私政策**：填 URL，并在小米后台**勾选"已收集的数据类型"**（设备标识、网络信息、用户输入）。
4. **第三方 SDK 列表**：小米要求在隐私政策中**逐项列出**第三方 SDK（如阿里云语音 SDK、uni 统计、微信支付等），含 SDK 名称、收集数据、用途、官方隐私政策链接。
5. **教育类资质核查**：本项目为志愿填报工具，**不属于学科类校外培训**，无需办学许可证；如审核问询，回复说明"非 K12/学科类培训，不涉及未成年人在线授课"。

#### 2.2.5 小米特殊审核点

| 检查点 | 要求 |
|--------|------|
| 64 位适配 | 强制 `arm64-v8a`（已满足） |
| targetSdkVersion | ≥33（项目 34，已满足） |
| HyperOS 适配 | 通知权限申请、深色模式不要求强制支持但需不崩溃 |
| 教育资质 | 非学科类培训需明确说明 |
| 应用名查重 | "[你的应用名称]"如已被注册需提交商标证或更换 |

#### 2.2.6 提交审核

填完所有信息 → "提交审核" → 通常 1-2 个工作日出结果 → 通过后默认全渠道发布。

### 2.3 VIVO 应用商店

#### 2.3.1 基本信息

| 项 | 值 |
|------|-----|
| 开发者后台 | https://dev.vivo.com.cn/ |
| 上传 APK | `[你的appid]__1.0.0_VIVO.apk` |
| 审核周期 | 1-3 个工作日 |
| 账号类型 | 企业开发者 |
| 适配系统 | OriginOS / Funtouch OS |
| 关联市场 | **iQOO 应用商店共用同一开发者账号**（详见 §3.2） |

#### 2.3.2 注册开发者账号

1. 访问 https://dev.vivo.com.cn/ → "注册账号"（手机号注册）。
2. 进入"开发者中心" → "实名认证" → 选"企业开发者"。
3. 提交：营业执照、法人身份证正反面、对公账户、加盖公章的"VIVO 开发者授权书"。
4. VIVO 人工审核（1-3 个工作日），同时完成对公打款验证。
5. 认证通过后即可同时分发到 **VIVO 应用商店 + iQOO 应用商店**（同一控制台）。

#### 2.3.3 创建应用并上传

1. 控制台 → "应用" → "提交新应用" → "Android"。
2. 上传 `_VIVO.apk`，系统解析包名 `com.yourcompany.app`。
3. 应用名称：[你的应用名称]；分类：教育-学习；适用人群：全年龄。
4. 上传素材：
   - 图标 512×512 PNG；
   - 截图 ≥4 张（1080×1920，竖屏）；
   - 海报 1 张（1080×1920，可选）。
5. 简介与描述：复用华为手册 §3.3 文案。

#### 2.3.4 资质与合规

1. **软件著作权**：必填扫描件，**VIVO 强校验软著名称与应用名称一致性**，不一致须提供简称证明或更换名称。
2. **ICP 备案**：必填，并需上传备案截图（工信部备案查询页截图）。
3. **隐私政策**：必填 URL，且应用内首启隐私弹窗的文案需与提交的隐私政策一致。
4. **测试账号**：必填，VIVO 审核会真机走完整流程。

#### 2.3.5 VIVO 特殊审核点

| 检查点 | 要求 |
|--------|------|
| 软著一致性 | 软著名 vs 应用名 必须一致或有简称证明 |
| ICP 备案截图 | 需上传工信部备案截图（不只是备案号） |
| 推送权限 | 如使用 VIVO Push SDK，需另外申请推送权限（本项目暂未集成可忽略） |
| OriginOS 兼容 | 建议在 OriginOS 4 真机回归 |
| 教育类应用 | 与小米一致，志愿填报工具非学科类培训，无需办学许可 |

#### 2.3.6 提交审核

提交后 1-3 个工作日出结果，通过后**自动同时上架 VIVO 应用商店与 iQOO 应用商店**（无需重复提交）。

## 三、其他小型市场补充

### 3.1 一加（OnePlus，与 OPPO 共用）

- **关键事实**：一加自 2021 年与 OPPO 全面合并，**一加应用商店现已并入 OPPO 软件商店**，无独立开发者后台。
- **上架方式**：在 OPPO 开放平台（§2.1）上架一次即可，分发渠道**自动覆盖一加机型 + ColorOS / OxygenOS**。
- **APK 文件**：使用 `_OPPO.apk` 即可，**无需额外打一加渠道包**。
- **OxygenOS / ColorOS 12+ 已统一内核**：兼容性测试覆盖 OPPO 即可视为覆盖一加。

### 3.2 iQOO（与 VIVO 共用）

- **关键事实**：iQOO 是 VIVO 子品牌，**iQOO 应用商店与 VIVO 应用商店共用同一后台**（dev.vivo.com.cn）。
- **上架方式**：在 VIVO 开放平台（§2.3）提交后，自动同步分发到 iQOO 应用商店。
- **APK 文件**：使用 `_VIVO.apk`。
- **OriginOS** 兼容即同时覆盖 iQOO 全系机型。

### 3.3 腾讯应用宝

| 项 | 值 |
|------|-----|
| 开发者后台 | https://open.qq.com/ |
| 上传 APK | `[你的appid]__1.0.0_应用宝.apk` |
| 审核周期 | 1-5 个工作日（流程偏严） |
| 账号类型 | 企业开发者，需对公打款验证 |

**特殊要求**：
1. 软著名称必须与应用名严格一致；
2. 需上传企业 LOGO（用于商家中心）；
3. 隐私政策中需声明所有第三方 SDK；
4. 教育类需说明非学科培训；
5. 提供完整测试账号 + 必要时录制功能演示视频；
6. 国家计算机病毒应急中心病毒扫描（应用宝自动跑）。

**操作步骤简版**：注册 QQ → 开放平台实名认证 → 创建应用 → 上传 `_应用宝.apk` → 填资质 → 提交审核。

## 五、常见被拒原因与对策

### 5.1 隐私合规类（占被拒 60%）

| 问题 | 对策 |
|------|------|
| 同意前已发起网络请求 | 隐私弹窗（App.vue 已实现）同意前**禁止任何 uni.request / SDK 初始化** |
| 隐私政策未列第三方 SDK | 隐私政策须含：阿里云语音 SDK、uni 统计、微信支付（如有）等的名称/数据/用途/官方政策链接 |
| 拒绝权限后崩溃或强退 | 拒绝非必要权限后必须能继续使用核心功能 |
| 一启动就申请所有权限 | 必须在使用到时再动态申请（permission.js 已实现） |
| 注销账号入口缺失 | 必须保留 我的-设置-注销账号 入口 |

### 5.2 资质类

| 问题 | 对策 |
|------|------|
| 软著名 vs 应用名不符 | 提供简称证明，或使用软著上的全称 |
| ICP 备案查不到 | 等待工信部最终展示（备案通过后 24 小时内同步） |
| 包名与备案不一致 | 必须严格使用 `com.yourcompany.app`，与备案一致 |

### 5.3 内容/UI 类

| 问题 | 对策 |
|------|------|
| 文案含"最/第一/国家级" | 改为"智能/科学/专业"等中性词 |
| 截图含其他品牌外观 | 使用纯净状态栏或本品牌截图 |
| 应用名查重 | 商标查询后注册，必要时提供商标受理通知书 |

### 5.4 兼容性类

| 问题 | 对策 |
|------|------|
| MIUI/HyperOS 通知不弹 | 申请 POST_NOTIFICATIONS 权限并测试 |
| ColorOS 字体放大变形 | 使用 rpx 单位，避免固定 px |
| OriginOS 弹窗被拦截 | 检查白屏/弹窗是否被系统拦截，给用户引导 |

## 六、推进时间表与策略

### 6.1 推荐推进顺序

```
Day 1-3：注册四大市场企业开发者账号 + 准备材料（截图/描述/测试账号）
Day 4：HBuilderX 一次打包出全部渠道包
Day 5：先提华为（最严，过审后其他基本都能过）
Day 6：华为审核期间同步提小米（最快 1-2 天）
Day 7：提交 OPPO（自动覆盖一加）+ VIVO（自动覆盖 iQOO）+ 荣耀
Day 8-10：审核期间提应用宝、360、三星等小市场
Day 11+：所有市场全面上线
```

### 6.2 后续版本同步发布节奏

1. 修改 `manifest.json`：递增 `versionCode`（如 100→101），可选递增 `versionName`；
2. HBuilderX 一次打包输出全部渠道包；
3. 各市场后台→ 版本管理→ 新建版本→ 上传对应渠道包→ 填写更新说明→ 提交；
4. **首次上架完成后**可启用 uni-app 一键发布（参见 [`uniapp一键发布指南`](./uniapp一键发布指南.md)）实现自动化分发。

### 6.3 时间预估

| 阶段 | 耗时 |
|------|------|
| 四大市场 + 小市场 全部账号注册 | 3-5 天 |
| 材料准备 | 1-2 天 |
| 打包 | 0.5 天 |
| 各市场提交 | 1 天 |
| 审核等待（最长以三星/应用宝为准） | 1-7 天 |
| **首次全量上架合计** | **约 7-15 天** |
