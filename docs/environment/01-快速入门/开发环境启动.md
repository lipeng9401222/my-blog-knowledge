## 模式介绍
- 从F10开始强绑定vue开发， 版本特性和新的主题均采用vue技术栈。仅F9.5.3+上vue是可选；低于F9.5.3则只支持老的Mini模式
- mini开发仅需要启动后端工程，vue则需要额外启动独立的vue-web。

###  修改项目基础路径和虚拟路径
- 文件位置：<font style="color:red">`src/config.js`</font>
- 修改项目基础路径：调整里面的<font style="color:red">`BASEPATH`</font>的配置即可，该路径决定打包后页面里的路径前缀(如果希望打包后可以不带应用路径访问，<font style="color:red">`BASEPATH_PRODUCTION`</font>请配置为<font style="color:red">`/`</font>)
``` js
// 项目基础路径,研发环境取用`/`,正式环境需与`ROOTPATH`同步
const BASEPATH = '/';
```

- 修改项目虚拟路径：调整里面的<font style="color:red">`ROOTPATH`</font>的配置即可，该路径用于访问后端接口或页面
``` js
// 项目虚拟路径
const ROOTPATH = '/epoint-web';
```


### 修改vite代理配置(仅源码启动涉及，如要打包，不需要动vite配置)
- 文件位置：<font style="color:red">vue-web工程的`vite.config.js`</font>
- 注意：如果是组件化子工程，它的`vite.config.js`不用配置，目前必须通过主vue-web工程启动，即使强行启动也只能看个模板框架，并不能实际运行。
- 代理配置说明：参考[vite请求代理](http://192.168.219.170/docs/vue/latest/frame/guide/developer-guide.html#%E8%AF%B7%E6%B1%82%E4%BB%A3%E7%90%86)
- 代理规则语法：参考[vite:server-proxy](https://www.vitejs.net/config/#server-proxy)
- 需要注意的是，如果有多个不同contextPath的后端，需要配多组server,一般情况下只需要一个；proxy配置请参考以下代码：<br>

```js
//标准形式参考：
server: {
    proxy: {
      // 代理 API 请求
      '/epoint-web': {
        target: 'http://localhost:8080',
        changeOrigin: true
      },
      // '/EpointFrame': {
      //   target: 'http://127.0.0.1:8081',
      //   changeOrigin: true
      // }
    }
  }

//实际已经经过封装，只需要修改最开始的后端服务的运行地址，最后一段指向到后端
const BACKEND_SERVER_URL = Config.isMock ? 'https://fe.epoint.com.cn/mock/752/eui-vue/' : 'http://localhost:8080';

```

### 启动vue
- 如果是vue组件化模式(推荐)：命令行<font style="color:red">打开到pnpm workspace工程目录下</font>，按以下命令执行：

``` bash
# 请先确保install过了，否则请先执行 pnpm install, 或简写 pnpm i
# 如果需要确保一些版本号填的tag的组件能够同步更新，需要执行 pnpm up && pnpm install
pnpm install
# 如果子工程有改动，需要将子工程进行build,通过`eui-cli build`一次将所有组件全部build
eui-cli build

# cd到pnpm工作区的 vue-web目录下(vue-web需要替换为实际的工程名)，需要到web目录下才能启动
cd vue-web

# 启动执行
pnpm run dev

```
