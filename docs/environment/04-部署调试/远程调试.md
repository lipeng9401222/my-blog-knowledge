## 后端Java连接Tomcat调试
有的时候，我们直接拿了项目的部署包过来，放在本地tomcat里面跑程序，但是无法调试框架代码，所以我们不得不在框架里面写入日志然后在打包放到tomcat下面去执行，排查问题很不方便。

针对上述情况，有下面的解决方案，可以让我们在tomcat下跑也能对框架代码进行调试：

### 远程服务器配置（以ecloud 为例）

#### 1. 挂载wrapper.conf配置

![](https://fdoc.epoint.com.cn/onlinedoc/rest/frame/base/attach/attachAction/getContent?isCommondto=true&attachGuid=b7a76f93-cc40-4604-a7ad-cbf2d220a6b2)

#### 2. 添加调试参数

在挂载的wrapper.conf配置文件中最下面加入以下参数：

```
# 开启远程Debug，设置监听端口
wrapper.java.additional.26=-agentlib:jdwp=transport=dt_socket,server=y,suspend=n,address=*:50022
```

参数说明：

• transport=dt_socket：使用套接字传输

• server=y：作为调试服务器

• suspend=n：启动时不暂停（若设为 y 则会等待调试器连接才启动应用）

• address=*:50022：监听 50022 端口（* 表示所有接口，可改为具体 IP 限制访问）

#### 3. 重启应用

修改保存后，重启 容器 使配置生效。

#### 4. 验证端口监听

使用以下命令检查远程服务器是否在监听 50022 端口：

```
netstat -tunp|grep 50022
```

### IDE配置

#### IntelliJ IDEA 配置

![](https://fdoc.epoint.com.cn/onlinedoc/rest/frame/base/attach/attachAction/getContent?isCommondto=true&attachGuid=82b25e56-098a-4d5f-ad79-4aea73e64b76)

• 打开项目，点击顶部菜单 Run → Edit Configurations

• 点击左上角 + 号，选择 Remote JVM Debug

• 填写配置信息：

• Name：自定义名称（如 Remote Tomcat Debug）

• Host：远程服务器 IP 地址

• Port：调试端口（与服务器配置一致，默认为 5005）

• Debugger mode：选择 Attach to remote JVM

• Transport：选择 Socket

#### eclipse配置

- 打开eclipse，在对应版本框架代码的工程上进行，选中 debug as 出现如下界面，并按照图中进行配置

#### IntelliJ IDEA 操作

##### 1. 设置断点

• 在 IDEA 右上角选择刚创建的远程调试配置

### 调试技巧

#### 1. 常用调试操作

• F8：Step Over（跳过当前行）

• F7：Step Into（进入方法内部）

• Shift + F8：Step Out（跳出当前方法）

• Alt + F9：Run to Cursor（运行到光标位置）

• Alt + F8：Evaluate Expression（计算表达式）

#### 2. 变量查看

• 在调试窗口的 Variables 面板查看当前变量值

• 右键变量选择 Watch 可添加到监视列表

• 使用 Drop Frame（调试工具栏回退箭头）可回退到上一个调用栈

#### 3. 条件断点

• 右键断点，设置条件表达式（如 id == 100）

• 仅当条件满足时触发断点，减少不必要的中断

### 常见问题及解决

#### 1. 无法连接到远程调试端口

• 检查服务器防火墙是否开放 50022 端口

```
firewall-cmd --zone=public --add-port=50022/tcp --permanent
```

• 验证服务器监听状态（使用netstat命令）

• 检查本地与远程网络连通性（使用 ping 和 telnet 命令）

#### 2. 断点未命中

• 确保本地与远程代码完全一致

• 检查类路径是否相同（特别是第三方依赖）

• 尝试重启远程应用和调试会话

#### 3. 调试过程中应用响应缓慢

• 调试会显著降低应用性能，仅在必要时使用

• 避免在生产环境长时间运行调试会话

• 考虑使用条件断点减少不必要的中断

### 安全注意事项

1. 调试端口（默认 5005）可能存在安全风险，建议：

• 仅在测试 / 开发环境使用远程调试

• 生产环境如需使用，限制调试端口只对可信 IP 开放

• 调试完成后及时关闭调试配置

1. 不要在调试会话中执行未经验证的代码或表达式

2. 确保调试会话不会影响生产业务

### 扩展配置

#### 1. 自定义调试端口

若默认端口被占用，可修改服务器和 IDEA 配置中的端口号（如改为 8000）

#### 2. 远程 Tomcat 免重启调试

使用 suspend=y 参数可让应用启动时暂停，等待调试器连接

```
-agentlib:jdwp=transport=dt\_socket,server=y,suspend=y,address=\*:50022"
```
