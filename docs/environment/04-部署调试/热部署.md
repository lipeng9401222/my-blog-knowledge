## 启用jrebel
代码修改之后的热加载，主要通过jrebel插件来实现。

![20240801-f07ad826.png](https://fdoc.epoint.com.cn/onlinedoc/rest/frame/base/attach/attachAction/getContent?isCommondto=true&attachGuid=d1067ccb-c88a-4e8c-b80d-c6be8d0e7572)

使用JRebel 插件方式生成 rebel.xml，框架的pom中默认带了这个插件，如果没有请手动添加

![20240801-ff80b4dc.png](https://fdoc.epoint.com.cn/onlinedoc/rest/frame/base/attach/attachAction/getContent?isCommondto=true&attachGuid=f9073c28-49de-45dc-8e82-7fec1dad7c1a)

启动时看到日志中输出如下信息说明 JRebel 监控成功

![20240801-506d46d0.png](https://fdoc.epoint.com.cn/onlinedoc/rest/frame/base/attach/attachAction/getContent?isCommondto=true&attachGuid=81904479-0798-4a31-afdb-441c2b67cd0d)

启动后修改Java源码文件，则会输出以下信息：

```java
[2016-03-18 21:14:14] JRebel: Reloading class 'com.epoint.core.dao.CommonDao'.
```

## 动态刷新
- 配置文件的修改，通过刷新小工具来实现。
- 刷新小工具默认已经内置在框架中，如何集成、如何登录使用，可以参考[[小工具使用说明](https://fdoc.epoint.com.cn/onlinedoc/rest/d/JFbuqy "小工具使用说明")](?file=080-效能工具/030-自检工具/030-部署指南)。

### properties文件动态加载
所有properties文件的修改，都可以通过小工具的[[一键刷新](https://fdoc.epoint.com.cn/onlinedoc/rest/d/jmQNfi "一键刷新")](?file=080-效能工具/030-自检工具/020-功能介绍/060-小工具#缓存刷新工具)功能完成动态加载，功能可以覆盖如下应用场景：修改安全配置、修改免登录、修改数据库连接、修改其他业务配置等等。

![image-20220615174154174.png](https://fdoc.epoint.com.cn/onlinedoc/rest/frame/base/attach/attachAction/getContent?isCommondto=true&attachGuid=0d408ec6-8711-4965-ba43-aa8d7ea214b4)

9.3.3+框架版本，系统登录后，可以发现多出悬浮刷新工具条，点击倒三角展开工具条，点击小工具刷新也可以实现类似功能。

![20240805-6dfce8ff.png](https://fdoc.epoint.com.cn/onlinedoc/rest/frame/base/attach/attachAction/getContent?isCommondto=true&attachGuid=c908f460-2147-4d3d-bf9f-f7c4faa70f9f)


### log4j2.xml文件动态加载
log4j2.xml的修改，可以通过小工具的[[日志级别配置](https://fdoc.epoint.com.cn/onlinedoc/rest/d/jmQNfi "日志级别配置")](?file=080-效能工具/030-自检工具/020-功能介绍/060-小工具#日志级别配置)功能完成动态加载。
