# 06 学习资源与官方文档

## 使用原则

资源不在多，而在能闭环。每个主题只保留最优先的资料入口，并配合一个实操任务。

推荐顺序：

1. 先读官方 Getting Started。
2. 跟敲一个最小示例。
3. 改造成工业场景练习。
4. 写一页学习笔记。
5. 放进项目作品。

## 第 1 类：Python 和工程基础

### 必读资料

| 资料 | 怎么读 |
|---|---|
| [Python Tutorial](https://docs.python.org/3/tutorial/) | 重点读数据结构、模块、输入输出、错误和异常 |
| [Python Standard Library](https://docs.python.org/3/library/) | 遇到 `csv`、`json`、`pathlib`、`logging` 时查 |

### 必做练习

1. 读取一个 CSV 文件。
2. 统计每列缺失率。
3. 输出 JSON 格式的数据质量报告。
4. 使用 logging 记录处理步骤。
5. 用 argparse 接收文件路径参数。

### 学习笔记模板

```markdown
# Python 本周笔记

## 学到的 3 个语法点

## 写出的 2 个脚本

## 遇到的 1 个报错

## 下周要改进的代码习惯
```

## 第 2 类：数据处理

### 必读资料

| 资料 | 怎么读 |
|---|---|
| [pandas Getting Started](https://pandas.pydata.org/docs/getting_started/) | 重点读 10 minutes to pandas、IO、groupby、time series |
| [NumPy Absolute Beginners](https://numpy.org/doc/stable/user/absolute_beginners.html) | 重点读数组、索引、统计函数 |

### 必做练习

1. 生成 10 万行模拟传感器数据。
2. 统计缺失率和异常值比例。
3. 按设备和小时聚合。
4. 构造滑窗均值、标准差、最大值。
5. 输出清洗前后对比图。

### 推荐主题

| 主题 | 学到什么程度 |
|---|---|
| `read_csv` | 会处理编码、时间字段、缺失值 |
| `groupby` | 能按设备、班次、日期聚合 |
| `merge` | 能合并传感器数据和故障记录 |
| `resample` | 能按固定时间频率对齐 |
| `rolling` | 能构造滑窗特征 |

## 第 3 类：传统机器学习

### 必读资料

| 资料 | 怎么读 |
|---|---|
| [scikit-learn Getting Started](https://scikit-learn.org/stable/getting_started.html) | 先掌握 fit、predict、score |
| [scikit-learn User Guide](https://scikit-learn.org/stable/user_guide.html) | 按任务查分类、回归、异常检测 |
| [scikit-learn Model Evaluation](https://scikit-learn.org/stable/modules/model_evaluation.html) | 重点看分类和回归指标 |

### 必做练习

1. 分类：预测设备是否故障。
2. 回归：预测未来温度或质量指标。
3. 异常检测：识别异常工况。
4. 对比至少 3 个模型。
5. 写一页模型评估报告。

### 推荐模型顺序

| 顺序 | 模型 | 原因 |
|---:|---|---|
| 1 | 规则阈值 | 工业现场常用，便于解释 |
| 2 | Logistic/Linear Regression | 可解释 baseline |
| 3 | Random Forest | 稳健，适合表格数据 |
| 4 | Gradient Boosting | 性能较强 |
| 5 | Isolation Forest | 异常检测入门 |

## 第 4 类：深度学习和时序预测

### 必读资料

| 资料 | 怎么读 |
|---|---|
| [PyTorch Tutorials](https://pytorch.org/tutorials/) | 重点看 tensors、autograd、neural networks、data loading |
| [TensorFlow Tutorials](https://www.tensorflow.org/tutorials) | 可作为补充，不必和 PyTorch 同时深学 |

### 必做练习

1. 用 PyTorch 训练 MLP。
2. 自定义 Dataset 读取时序窗口。

4. 训练简化 Transformer Encoder 做对比。
5. 输出预测曲线和误差分析。

### 学习建议

优先 PyTorch。TensorFlow 只需要达到“看得懂、能复现简单例子”的程度，除非目标公司明确要求 TensorFlow。

## 公开数据集建议

| 数据集 | 用途 |
|---|---|
| AI4I 2020 Predictive Maintenance Dataset | 预测性维护入门 |
| NASA Turbofan Engine Degradation Dataset | 剩余寿命预测 |
| Tennessee Eastman Process Dataset | 化工过程故障检测 |
| MIMII Dataset | 工业机器声音异常检测 |

使用建议：

1. 第一阶段可先用模拟数据，降低上手难度。
2. 第二阶段再使用公开数据集，提高真实性。
3. 每个数据集都要写字段说明和业务背景。

## 优先不用深挖的资源

这些内容不是完全不学，而是不要在前 3 个月占用主线时间：

| 内容 | 暂缓原因 | 什么时候再补 |
|---|---|---|
| 大模型预训练和微调细节 | 岗位主线是应用和集成 | 完成 RAG 和 API 项目后 |
| Kubernetes | 当前路线只要求本地 Docker 交付 | FastAPI + Docker 熟练后 |
| 复杂强化学习 | 与工艺优化和排产有关系，但入门成本高 | 综合项目完成后作为加分项 |
| 工业视觉高阶检测 | 岗位截图主线更偏数据和 RAG | 主线作品集完成后拓展 |
| 复杂数学推导 | 容易拖慢项目产出 | 遇到模型边界时按需补 |

## 公开数据集使用顺序

| 阶段 | 推荐数据 | 使用目标 |
|---|---|---|
| 第 1 到 4 周 | 自己生成模拟传感器数据 | 降低上手难度，练习清洗和统计 |
| 第 5 到 10 周 | AI4I 2020 Predictive Maintenance Dataset | 做分类、回归、异常检测 baseline |
| 第 11 到 16 周 | NASA Turbofan Engine Degradation Dataset | 做剩余寿命或时序预测对比 |
| 第 17 到 22 周 | 自编工业规程 Markdown/TXT | 做 RAG 文档清洗、切分、引用来源 |
| 第 28 周以后 | Tennessee Eastman Process Dataset | 补流程行业故障检测和数据流理解 |

## 每个资源的正确打开方式

不要这样学：

1. 收藏 100 个链接。
2. 看很多视频但不写代码。
3. 复制教程项目但不改造。
4. 学新工具逃避主线。

应该这样学：

1. 只选 1 个官方入口。
2. 跑通最小示例。
3. 改成工业场景。
4. 写 README。
5. 放入作品集。
