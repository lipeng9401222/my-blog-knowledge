# Skills 收集

用于沉淀高价值的 `skills`、可复用提示词套路、自动化流程模板与能力清单。

`Skill` 更适合理解为“可被稳定触发的一段能力说明书”，它通常同时包含触发条件、上下文收集方式、执行步骤、输出要求和校验规则。

## 推荐阅读

- [AI Coding Skills 体系设计：从 Prompt、上下文到可复用工作流](/ai/skills/ai-coding-skills-system-design)

## 可以先区分的 4 类内容

- 基础技能：例如代码阅读、Bug 排查、文档整理、接口分析
- 领域技能：例如前端性能优化、VitePress 站点维护、Git 协作规范
- 工作流技能：例如“先分析后改动”“改完自动回归”“产出带检查清单”
- 知识增强技能：例如结合项目约定、目录结构、组件规范做定制化输出

## 外部参考

- [skills.sh](https://skills.sh/)：偏向 AI Agent Skills 的目录、排行榜与发现入口，适合找灵感，不适合作为团队规范的直接替代。

## 规划内容

- 开发类 skills
- 文档整理类 skills
- 数据处理类 skills
- 团队知识沉淀类 skills
