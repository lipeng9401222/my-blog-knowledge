# PPT Agent 设计思路解析：从需求调研到 SVG 全自动生成

> 原文来源：[应该是目前最强的PPT Agent，附上完整思路分享](https://linux.do/t/topic/1782304)（LINUX DO 社区）

## 概述

这篇文章分享了一个完整的 **PPT Agent** 设计思路，核心理念是：让 AI 像专业顾问一样工作，而非"一键生成"。整个流程覆盖 **需求调研 → 资料搜集 → 大纲策划 → 策划稿 → 设计稿** 五个阶段，使用 Gemini 3 Flash 模型驱动，最终输出可编辑的 SVG 格式 PPT 页面。

## 核心工作流

整个 Agent 分为 4 个阶段，遵循专业 PPT 设计公司的工作模式：

### 阶段一：需求调研（忘了"一键生成"）

市面上的 AI PPT 工具普遍犯了一个错误——用户输入主题就立刻生成大纲和模板。但 **PPT 的灵魂是内容，不是皮囊**。

Agent 的第一步是像专业顾问一样做需求调研：

1. 先在网上检索相关资料
2. 向用户提出关键问题，厘清"为谁做？做什么？达到什么目的？"
3. 需求聊透后，结合资料生成真正"有的放矢"的大纲

大纲设计采用了 **便利贴法**——每一页的核心内容写成一张"数字便利贴"，逻辑结构一目了然，哪里不好就撕掉，顺序不对就调换。

### 阶段二：大量检索资料

大纲只是骨架，需要真实、准确的信息来填充。

- 工程方案：使用搜索接口自动检索
- 个人 DIY 推荐：使用 **Grok**，搜索和信息总结能力极强
- 用法：把大纲标题逐个丢给 Grok，自动搜集和整理所需资料

### 阶段三：生成策划稿

这是区别于普通 AI PPT 工具的关键步骤——在大纲和设计之间增加了一个 **策划** 环节。

灵感来源于国内顶尖 PPT 设计公司（报价 1万+/页）的工作流：他们有专门的"策划师"岗位，负责需求调研、资料检索、大纲规划，最终产出策划草稿给设计师，每页的位置、元素、版式都已固定。

**实践证明：先让 AI 生成简洁的页面初稿（策划稿），再去加设计效果，跑下来效果非常好。**

### 阶段四：卡片式布局生成设计

设计阶段采用 **卡片式布局（Bento Grid）**，这是 AI 最容易理解和掌握的设计语言。

选择卡片式布局的三大优势：

1. **能装**：一页可清晰承载大量信息
2. **灵活**：卡片数量、大小、位置随意组合，版式变化无穷
3. **AI 能懂**：AI 最容易理解和掌握的布局方式

最终输出 **整页 SVG 代码**，而非 HTML 或调用第三方库。

## 关键技术选型：为什么选 SVG

| 方案 | 优势 | 劣势 |
|------|------|------|
| SVG | 可导入 PPT 编辑、各设计软件兼容、无限放大不失真 | 工程复杂度高，需要从零摸索 |
| HTML | AI 生成更容易 | 无法直接导入 PPT 编辑 |
| 调用第三方库（如 Banana） | 开发成本低 | 灵活性和可编辑性受限 |

> SVG 兼容性最好（Office 2016+），支持无限放大，且完全可编辑。

## 核心 Prompt 分享

### Prompt 1：PPT 结构架构师

用于大纲规划阶段，基于金字塔原理设计 PPT 逻辑结构：

```markdown
# Role: 顶级的PPT结构架构师

## Profile
- 版本：2.0 (Context-Aware)
- 专业：PPT逻辑结构设计
- 特长：运用金字塔原理，结合**背景调研信息**构建清晰的演示逻辑

## Goals
基于用户提供的 **PPT主题** 和 **背景调研信息 (Context)**，设计一份逻辑严密、层次清晰的PPT大纲。

## Core Methodology: 金字塔原理
1. 结论先行：每个部分以核心观点开篇
2. 以上统下：上层观点是下层内容的总结
3. 归类分组：同一层级的内容属于同一逻辑范畴
4. 逻辑递进：内容按照某种逻辑顺序展开

## 重要：利用调研信息
你将获得一些关于主题的搜索摘要。请务必参考这些信息来规划大纲，
使其切合当前的市场现状或技术事实，而不是凭空捏造。

## 输出规范
请严格按照以下JSON格式输出，结果用[PPT_OUTLINE]和[/PPT_OUTLINE]包裹：

[PPT_OUTLINE]
{
  "ppt_outline": {
    "cover": {
      "title": "引人注目的主标题",
      "sub_title": "副标题",
      "content": []
    },
    "table_of_contents": {
      "title": "目录",
      "content": ["第一部分标题", "第二部分标题", "..."]
    },
    "parts": [
      {
        "part_title": "第一部分：章节标题",
        "pages": [
          { "title": "页面标题1", "content": [] },
          { "title": "页面标题2", "content": [] }
        ]
      }
    ],
    "end_page": {
      "title": "总结与展望",
      "content": []
    }
  }
}
[/PPT_OUTLINE]

## Constraints
1. 必须严格遵循JSON格式。
2. **页数要求**：{{PAGE_REQUIREMENTS}}
```

### Prompt 2：SVG 页面设计指令

用于设计生成阶段，将文字内容转化为 SVG 演示文稿页面：

```markdown
作为精通信息架构与 SVG 编码的专家，你的任务是将完整的文字内容转化为
一张高质量、结构化、具备高级感、简洁感和专业感的 SVG 演示文稿页面。要求如下：

1. 画布: SVG viewBox 必须是 0 0 1280 720。

2. 内容页的便当网格 (Bento Grid) 布局
这是一种灵活的网格系统，其布局应由内容本身的需求驱动，而非僵硬的模板。
通过组合不同尺寸的卡片，创造出动态且视觉有趣的布局。

- 核心原则:
  - 灵活性: 卡片数量不固定。可以是 1, 2, 3, 4, 5 或更多个，取决于如何更好地呈现信息。
  - 层级感: 使用卡片尺寸建立视觉层级。最重要的信息放在最大的卡片上。
  - 留白: 在所有卡片之间保持至少 20px 的间距。

- 布局组合示例:
  - 单一焦点: 一张大卡片覆盖大部分区域 (w=1200, h=580)。
  - 两栏布局:
    - 50/50 对称: 两张等宽的卡片。
    - 非对称: 一张较宽的卡片（2/3 宽度）用于主内容，一张较窄的（1/3 宽度）用于辅助信息。
  - 三栏布局: 三张等宽的卡片，适合并列比较三项内容。
  - 主次结合: 一张大的居中卡片，两侧各一张小的垂直卡片。
  - 顶部英雄式: 顶部一张宽幅"英雄"卡片，下方是 2-4 个较小的等宽卡片网格。
  - 混合网格 (自由度最高): 自由混合各种尺寸的卡片。

请你根据我的内容输出SVG代码，我的内容是：[此处填入内容]
```

## Bento Grid 布局规范

```
内容页的便当网格 (Bento Grid) 布局
核心原则:
- 灵活性: 卡片数量不固定，取决于如何更好地呈现信息
- 层级感: 使用卡片尺寸建立视觉层级，最重要的信息放在最大的卡片上
- 留白: 在所有卡片之间保持至少 20px 的间距

布局组合示例:
- 单一焦点: 一张大卡片覆盖大部分区域 (w=1200, h=580)
- 两栏布局: 50/50 对称 或 非对称 (2/3 + 1/3)
- 三栏布局: 三张等宽卡片，适合并列比较
- 主次结合: 一张大的居中卡片，两侧各一张小的垂直卡片
- 顶部英雄式: 顶部一张宽幅"英雄"卡片，下方 2-4 个较小的等宽卡片网格
- 混合网格: 自由混合各种尺寸的卡片，适应不同内容需求
```

## 架构总结

```
用户输入主题
    │
    ▼
┌─────────────────┐
│  1. 需求调研     │  AI 检索资料 + 向用户提问
│  2. 大纲策划     │  金字塔原理 + 便利贴法 → JSON 大纲
│  3. 策划稿生成   │  简洁页面初稿，确定版面规划
│  4. 设计稿生成   │  Bento Grid 布局 → SVG 代码输出
└─────────────────┘
    │
    ▼
可编辑的 SVG 页面（导入 PPT / 设计软件）
```

## 个人思考

- **工作流复用**：这套"调研→大纲→策划→设计"的流程，本质上是把人类专家的工作流编码为 AI Agent 的执行链，思路可迁移到其他内容生成场景
- **Prompt 分层**：将大纲规划和设计生成拆为两个独立 Prompt，降低了单次 Prompt 的复杂度，提升了各环节的可控性
- **SVG 选型**：牺牲工程复杂度换取可编辑性，是一个面向实际使用场景的务实选择
- **模型选择**：Gemini 3 Flash 在性价比和生成质量之间取得平衡，有条件可升级到 3.1 Pro
