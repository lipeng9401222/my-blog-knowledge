## 项目简历模板

### 项目 1：工业时序数据清洗与特征工程管道

简历版：

```text
工业时序数据清洗与特征工程管道
- 使用 Python、pandas、NumPy 构建传感器数据处理流程，支持时间戳解析、去重、缺失值处理、异常点标记和多频率重采样。
- 基于设备温度、压力、振动、电流等点位数据构造滑窗均值、标准差、最大值、变化率和滞后特征，输出可用于模型训练的数据集。
- 编写数据质量报告，统计缺失率、异常比例和字段分布，为预测性维护模型提供标准化数据输入。
```

面试 1 分钟讲法：

> 这个项目模拟 DCS/SCADA 传感器数据处理流程。我的目标是把原始设备时序数据变成可建模数据集。我主要做了时间戳清洗、缺失处理、异常标记、重采样和滑窗特征构造，并输出数据质量报告。这个项目对应岗位里的工业数据工程能力。

### 项目 2：设备预测性维护模型

简历版：

```text
设备预测性维护模型
- 基于工业设备时序特征构建故障风险预测任务，采用时间顺序切分训练集和测试集，避免未来数据泄漏。
- 对比 Logistic Regression、Random Forest、Gradient Boosting 与 LSTM 模型，从 Precision、Recall、F1 和混淆矩阵评估模型效果。
- 结合误报和漏报成本分析报警阈值，并输出风险等级和主要影响特征，用于模拟设备提前预警。
```

面试 1 分钟讲法：

> 这个项目解决的是设备故障提前预警。我用过去一段时间的温度、压力、振动、电流等滑窗特征预测未来是否有故障风险。评估时我没有只看准确率，而是重点看召回率和漏报，因为工业故障漏报成本更高。

### 项目 3：工业规程 RAG 知识库

简历版：

```text
工业规程 RAG 知识库
- 基于 LangChain/LlamaIndex 构建工业规程文档问答流程，完成文档清洗、分块、向量化、检索和大模型回答。
- 为每个文档 chunk 保留文件名、段落编号等 metadata，实现答案引用来源返回，降低规程问答场景中的幻觉风险。
- 构建 30 条问题评测集，对检索命中率、回答正确率和失败案例进行分析，并优化 chunk 大小、top_k 和提示词模板。
```

面试 1 分钟讲法：

> 这个项目是把设备手册和操作规程做成知识库问答。核心不是简单接一个聊天机器人，而是把文档清洗、切分、向量化、检索和引用来源串起来。对于工业规程类问题，我特别强调答案必须可追溯，检索不到依据时不能让模型乱答。

### 项目 4：模型服务与工业平台集成 Demo

简历版：

```text
模型服务与工业平台集成 Demo
- 使用 FastAPI 将预测性维护模型和工业 RAG 知识库封装为 RESTful API，提供 `/health`、`/predict`、`/rag/query` 等接口。
- 使用 Pydantic 完成请求体验证，统一异常处理和日志记录，返回风险分数、风险等级、预测原因和知识库引用来源。
- 编写 Dockerfile 完成本地容器化部署，模拟 MES/工业互联网平台通过 HTTP 调用模型服务。
```

面试 1 分钟讲法：

> 这个项目把前面的模型和知识库变成可调用服务。我用 FastAPI 提供健康检查、预测和知识库问答接口，用 Docker 固化运行环境，模拟工业平台通过 HTTP 调用 AI 服务。它对应岗位中的模型服务集成和部署能力。

3. 缺失值有哪些处理方法？工业数据中怎么选择？
4. 异常值一定要删除吗？为什么？
5. 什么是标签泄漏？时序预测中如何避免？
6. 如何构造滑窗特征？
7. 10 万行和 1000 万行数据处理有什么差异？
8. 如何写一个可复用的数据清洗脚本？

## 面试回答原则

### 不要这样回答

```text
我用了 Random Forest，效果还可以。
```

### 应该这样回答

```text
我先用规则阈值和 Logistic Regression 做 baseline，再用 Random Forest 提升非线性表达能力。评估时重点看召回率和混淆矩阵，因为故障预测中漏报成本高于误报。最后我根据业务可接受的误报率调整了报警阈值。
```
