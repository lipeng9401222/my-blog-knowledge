# 华为应用市场（AppGallery）上架指南

> 适用项目：uni-app 构建的 Android App
> 开发主体：你的公司名称

---

## 一、前期准备

### 1.1 注册华为开发者账号

1. 访问 [华为开发者联盟](https://developer.huawei.com/consumer/cn/)
2. 点击"注册"，选择**企业开发者**
   - 需要：营业执照、对公银行账户、法人身份证
   - 主办单位：你的公司名称
3. 完成实名认证（审核通常 1-3 个工作日）
4. 登录 [AppGallery Connect](https://developer.huawei.com/consumer/cn/service/josp/agc/index.html)

### 1.2 创建应用

1. 登录 AppGallery Connect → 点击"我的应用"
2. 点击"新建应用"
3. 填写信息：
   - **应用名称**：你的应用名称
   - **应用分类**：应用 → 对应分类
   - **默认语言**：中文（简体）
   - **应用包名**：`com.yourcompany.app`
   - **设备类型**：手机
   - **应用类型**：选择 **Android**（不是 HarmonyOS！见下方说明）

### 1.3 ⚠️ Android vs 鸿蒙：选哪个？

华为应用市场支持两种应用类型，**本项目（uni-app 构建的 APK）必须选 Android**：

| 类型 | 包格式 | 适用系统 | 适用机型 | 用户体量 |
|------|--------|----------|----------|----------|
| **Android** ✅本项目 | `.apk` / `.aab` | EMUI / HarmonyOS 2.x~4.x（兼容Android） | Mate 50/P50/P60/Nova等绝大多数机型 | **数亿设备** |
| HarmonyOS（鸿蒙原生） | `.hap` / `.app` | HarmonyOS NEXT（5.0+，纯鸿蒙） | Mate 60/70 部分新机 | 较少（新增中） |

**结论**：
- ✅ 选 **Android** 即可覆盖华为绝大多数用户。这是 uni-app/Flutter/React Native 项目上架华为的**主流且正确**做法。
- ❌ 不要选 HarmonyOS——鸿蒙原生需要用 ArkTS/DevEco Studio 重新开发，与本项目技术栈不兼容。
- 鸿蒙原生版本属于**未来可选项**，不是必选；可以先把 Android 版本上架后再规划。

两种类型在 AppGallery Connect 里**互不冲突**，未来如果要做鸿蒙原生版，可以在同一应用下追加 HarmonyOS 类型。

### 2.5 渠道包选择与多市场打包策略

> 适用场景：本项目需同时上架 **华为 / 荣耀 / OPPO / 小米** 四个应用市场。

#### 2.5.1 渠道包的作用

渠道包是在 APK 中嵌入一个"渠道标识"，用于在 uni 统计或自有埋点系统中**区分用户来自哪个应用市场**。

- 它**本身不影响审核**——不勾任何渠道也能上架，只是无法区分来源数据。
- 勾选多个渠道时，HBuilderX 会**一次打包输出多个 APK**，每个带不同渠道标识。

#### 2.5.2 四个目标市场对应关系

| 应用市场 | HBuilderX 渠道包选项 | 上传哪个 APK |
|---------|---------------------|--------------|
| 华为应用市场 | ☑ 华为应用商店 | `_华为应用商店.apk` |
| 小米应用商店 | ☑ 小米应用商店 | `_小米应用商店.apk` |
| OPPO 软件商店 | ☑ OPPO | `_OPPO.apk` |
| **荣耀应用市场** | ❌ 列表无该选项 | ☑ **"无"**（通用包） |
| VIVO / 应用宝 / 360 / GooglePlay | 本次不上架 | 不勾 |

> ⚠️ 荣耀（HONOR）是从华为分拆后的独立品牌，HBuilderX 暂未内置荣耀渠道。直接用勾选"无"打出的通用包上传到荣耀开发者服务即可。

#### 2.5.3 推荐勾选方案（一次打包覆盖四市场）

**渠道包**：
- ☑ 无（通用包，用于荣耀）
- ☑ 华为应用商店
- ☑ 小米应用商店
- ☑ OPPO
- ☐ GooglePlay(AAB) / 应用宝 / 360应用市场 / VIVO

**防重签**（强烈建议勾选，防止 APK 被破解后重签名上架）：
- ☑ 包名
- ☑ 证书
- ☐ appid（按需，开了会限制只能在指定 DCloud 账号下运行）

> ⚠️ **互斥规则**：勾选"证书"防重签后，HBuilderX 会弹窗提示"不支持使用安心打包，请切换到普通打包"。这是 HBuilderX 的固有限制，必须**二选一**：
> - 想要证书防重签 → 必须用**传统打包**（推荐，安全性优先）
> - 想要快速安心打包 → 必须**取消"证书"防重签**（仅保留"包名"）
>
> 正式上架建议保留证书防重签，遇到弹窗直接点"切换到普通打包并继续"。传统打包会上传源码及证书到 DCloud，但 DCloud 承诺打完不保留，对正式应用无实质风险。

**打包模式**：
- ⦿ 打正式包
- ○ 打自定义调试基座（仅开发调试用，**上架不要选**）

**附加选项**：
- ☐ 生成 iOS 符号表（dsym）—— Android 包不需要
- ☑ 生成 SourceMap —— 建议勾，方便 uni 统计的错误堆栈还原
- ☐ 对配置的 js/nvue 文件进行原生混淆 —— 本项目无该需求，不勾
- ☐ 广告联盟所有选项（腾讯优量汇 / 穿山甲 / 华为广告联盟 / 快手 / 百度 / Sigmob / Pangle / AdMob / unity 等全部）—— 本项目无广告变现，**全部不勾**
- ☐ 快手内容联盟等三方场景变现 SDK —— 不勾

**底部打包入口**：
- ⦿ **传统打包**（搭配证书防重签时**必选**，正式上架推荐）
- ○ 快速安心打包（仅在不勾"证书"防重签时可用）

#### 2.5.4 打包输出

按上述配置点击"打包"后，HBuilderX 会**一次性输出 4 个 APK**，文件名形如：

```
[你的appid]__1.0.0_general.apk            → 上传荣耀
[你的appid]__1.0.0_华为应用商店.apk         → 上传华为
[你的appid]__1.0.0_小米应用商店.apk         → 上传小米
[你的appid]__1.0.0_OPPO.apk                → 上传 OPPO
```

> 实际命名以 HBuilderX 输出为准，按渠道后缀对应上传即可。

#### 2.5.5 备选方案：单一通用包

如果暂时不需要做渠道统计，可以**只勾"无"**，打一个通用 APK，4 个市场都用同一个包上传。简单省事，但会损失渠道来源数据。

## 五、常见驳回原因与对策

### 5.1 隐私合规类（最常见）

| 问题 | 解决方案 |
|------|----------|
| 未经同意就采集用户数据 | 确保隐私弹窗同意前不发起任何网络请求（已实现） |
| 隐私政策内容不完整 | 补充数据收集清单、第三方SDK列表、数据存储/删除说明 |
| 权限申请时机不当 | 确保权限在使用时才申请，不要一启动就申请所有权限 |
| 拒绝权限后强制退出 | 拒绝非必要权限后应允许继续使用其他功能 |
| 隐私政策链接不可访问 | 确保 URL 在国内可正常访问 |

### 5.2 功能体验类

| 问题 | 解决方案 |
|------|----------|
| 应用闪退/崩溃 | 充分测试各机型兼容性 |
| 核心功能无法使用 | 确保测试账号可正常体验所有功能 |
| 加载过慢/无响应 | 添加 loading 状态，优化网络请求 |
| 存在空白页面 | 确保所有页面有内容或空状态提示 |

### 5.3 内容合规类

| 问题 | 解决方案 |
|------|----------|
| 截图与实际不符 | 使用真实应用截图 |
| 应用名称含违禁词 | 避免"最""第一"等绝对化用语 |
| 包含未备案的网络内容 | 确保服务器已完成ICP备案 |

### 5.4 兼容性类

| 问题 | 解决方案 |
|------|----------|
| 不适配全面屏 | 确保 targetSdkVersion >= 30 |
| 不适配深色模式 | 可以选择性支持或声明不支持 |
| 横竖屏切换异常 | 锁定竖屏方向 |

## 九、注意事项

1. **targetSdkVersion 要求**：华为应用市场要求 2024 年起新上架应用 targetSdkVersion >= 33，请确保满足
2. **64位支持**：当前 abiFilters 包含 `arm64-v8a`，已满足华为 64 位要求
3. **敏感权限**：`RECORD_AUDIO` 属于敏感权限，审核时会重点检查使用场景
4. **API 服务稳定性**：确保审核期间后端 API 服务正常运行
5. **测试账号**：建议准备一个已有数据的测试账号，让审核人员能看到完整功能

### 10.2 小米应用商店
