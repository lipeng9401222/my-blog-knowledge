# 02-创建数据库与账号权限

## 目的

创建一个业务数据库，并创建一个“给应用用”的数据库账号（避免应用长期使用 root）。

## 前置条件

- 你已经完成 [01-确认 MySQL 服务与版本](./01-确认MySQL服务与版本.md)
- 你能用管理员账号登录 MySQL（例如 root）

## 约定（建议统一）

你可以按下面默认值走，也可以替换成你自己的，但后续要保持一致：

- 数据库名：`epoint_dev`
- 应用账号：`epoint`
- 应用密码：`<DB_PASSWORD>`（你自己设置一个强密码，不要写进仓库）
- 字符集：`utf8mb4`
- 排序规则：`utf8mb4_general_ci`
 - 应用账号允许连接的 host：`localhost`（本机开发推荐）

## 操作步骤（命令行方式，推荐）

### 1. 登录 MySQL

```bash
mysql -uroot -p
```

### 2. 创建数据库

把 `<DB_NAME>` 替换成你的库名（建议 `epoint_dev`）：

```sql
create database if not exists `<DB_NAME>`
  default character set utf8mb4
  default collate utf8mb4_general_ci;
```

### 3. 创建应用账号并授权

把 `<DB_USER>` / `<DB_PASSWORD>` 替换成你的实际值：

```sql
create user if not exists '<DB_USER>'@'%' identified by '<DB_PASSWORD>';
grant all privileges on `<DB_NAME>`.* to '<DB_USER>'@'%';
flush privileges;
```

本机开发推荐只允许本机连接（更安全），把 host 设置为 `localhost`：

```sql
create user if not exists '<DB_USER>'@'localhost' identified by '<DB_PASSWORD>';
grant all privileges on `<DB_NAME>`.* to '<DB_USER>'@'localhost';
flush privileges;
```

### 4. 记录数据库连接信息

你需要记录：

- host：`127.0.0.1`
- port：`3306`
- db：`<DB_NAME>`
- user：`<DB_USER>`

## 预期结果

执行以下 SQL 能看到数据库存在：

```sql
show databases like '<DB_NAME>';
```

并能看到账号存在：

```sql
select user, host from mysql.user where user = '<DB_USER>';
```

## 常见问题与排查

### 1) `ERROR 1819 (HY000): Your password does not satisfy the current policy requirements`

说明你 MySQL 启用了强密码策略。

处理方式：

- 换一个更复杂的密码（长度更长、包含大小写+数字+特殊字符）

## 本步产出（你来填写）

- 数据库名：epoint_dev
- 应用账号：epoint
- 应用账号 host（% 或 localhost）：localhost
- 验证：`show databases like 'epoint_dev';` 返回 1 行（epoint_dev）
- 验证：`select user, host from mysql.user where user = 'epoint';` 返回 1 行（epoint / localhost）
