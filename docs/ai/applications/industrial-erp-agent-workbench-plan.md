# 工业互联网 / ERP Agent 工作台 · 完整产品规划
> 参考:CopilotKit(AG-UI 协议)、Generative UI 三种模式(Controlled / Declarative / Open-ended)、Shared State、Human-in-the-loop、Sub-agents。  
> 核心理念:Agent 不只吐文字,而是在运行时**动态生成并更新真正可交互的 UI 组件**,与前端**双向共享状态**,关键动作走**人工确认**。

## 阅读指南
本文档可直接用于立项、排期与拉团队,覆盖:背景机会、产品定位、用户与场景、功能全景、系统架构、Generative UI 设计规范、数据与集成、安全合规、里程碑排期、团队配置、风险与指标。前六节为「是什么/做什么」,第七节起为「怎么落地」。

## 〇、背景与机会
**行业痛点**:数据重(时序 + 业务数据分散多系统)、流程长(异常处置/单据审批跨部门)、出错代价高(一张错单可致停产或资损)、用户不爱敲字(一线更要「看得懂、点一下就完成」)。

**为什么现在做**:Agent 的多步推理与工具调用已可用于生产;AG-UI + Generative UI 让 agent 输出「活界面」而非纯文本,首次让重流程场景可用;企业普遍已有 ERP/MES,缺的是**智能操作层**。

**差异化**:市面 agent 平台大多停在「聊天框 + 工具调用」。真正的壁垒是把工作台做成\*\*「活的画布」\*\*:agent 边推理边长出可编辑组件,状态双向回流,高风险动作人工确认。
* * *
## 一、为什么是 Generative UI
工业互联网与 ERP 的共性痛点:**数据重、流程长、出错代价高、用户不爱敲字**。传统「聊天框 + 工具调用」解决不了。Generative UI 正好对症:
*   Agent 输出的是**活的组件**(表格 / 图表 / 表单 / 看板 / 时间轴),不是一段文字。
*   用户直接在组件上编辑,状态**回流**给 agent(Shared State)。
*   高风险动作(改单据、下工单、审批)强制**人工确认卡点**(Human-in-the-loop)。

* * *
## 二、产品定位
**不替换 SAP / 用友 / MES,而是做它们之上的 Agent 层**,通过 MCP / AG-UI 把老系统能力封装成工具调用。落地阻力最小,价值最快可见。

一句话定位:**面向工业互联网与 ERP 的生成式 UI Agent 工作台,让复杂系统「问一句、点一下」就完成闭环。**

### 目标用户角色

| 角色 | 关注点 | 典型动作 |
| ---| ---| --- |
| 一线工人/操作员 | 简单、快、少敲字 | 看告警、确认工单 |
| 班组长/车间主任 | 排产、派工、处置 | 审批工单、调排程 |
| 厂长/生产总监 | 全局 OEE、停机归因 | 看报表、问原因 |
| 财务/采购专员 | 单据准确、合规 | 开单、校验、审批 |
| IT/系统管理员 | 权限、审计、集成 | 配置、监控、回溯 |

统一范式:**自然语言输入 → agent 推理 + 拉数据 → 生成可交互组件 → 人工确认 → 回写系统 → 审计留痕。**

## 三、功能规划
### 3.1 工业互联网侧

| 功能 | 说明 | GenUI 模式 |
| ---| ---| --- |
| 设备异常「活报告」 | 检测到告警时生成带实时曲线(温度/振动/OEE)的诊断卡片 + 根因列表 + 一键下发工单 | Controlled |
| 自然语言问产线 | 「上周 3 号线为什么停机 6 次」→ 拉时序数据,渲染停机时间轴 + 归因图 | Declarative |
| 预测性维护 | Agent 建议换件时间,生成排程卡片,班组长在卡片上直接改时间/派人,状态回流 | Controlled + HITL |
| 数字孪生 overlay | Agent 输出组件叠在产线拓扑图上,异常节点高亮,点击下钻 | Open-ended |

### 3.2 ERP 侧

| 功能 | 说明 | GenUI 模式 |
| ---| ---| --- |
| 单据对话式生成与校验 | 「给宁波那批货开采购订单」→ 生成可编辑 PO 表单,自动带供应商/价格/税率,红框标异常项 | Controlled + HITL |
| 跨模块归因 | 「这个月毛利为什么降了」→ 串销售/采购/库存,渲染瀑布图拆解,逐段可下钻 | Declarative |
| 审批流智能卡点 | Agent 预审合规性,风险点做成受控组件推给审批人,低风险自动过、高风险强制人工 | Controlled + HITL |
| 主数据清洗助手 | 扫出重复/冲突物料主数据,生成对比合并界面,人一键裁决 | Controlled |

### 3.3 贯穿两侧的底座能力
1. **权限感知的 Generative UI**:同一查询,车间工人与厂长看到的组件粒度不同,agent 按角色渲染。生产环境刚需。
2. **强审计与可回滚**:每个 agent 动作(尤其改单据/下工单)全链路留痕、可回溯、可回滚。合规红线。
3. **Agent 执行可复现**:推理与工具调用存成 timeline,支持「从第 N 步重跑」「fork 分支」。把 agent 执行当 git 管理。
4. **离线/边缘兜底**:工厂网络不稳,工作台在边缘侧可降级运行,联网后同步。
5. **MCP 集成层**:对接现有 SAP/用友/MES,把老系统能力包成工具,不做替换。
6. **多 agent 编排可视化**:子 agent 分工/并行/handoff 全程可见,每个节点可人工干预。

## 四、MVP 建议
**先啃工业互联网的「异常诊断 → 一键工单」这一条闭环。** 场景窄、价值肉眼可见、演示效果炸,远比铺开做整个 ERP agent 靠谱。

闭环拆解:

1. 接入设备告警数据源(时序库 / MES)
2. Agent 诊断 + 生成诊断卡片(实时曲线 + 根因)
3. 人工确认 → 一键下发工单(回写 MES/工单系统)
4. 全链路审计留痕

## 六·五、内容生成能力:报表制作 & PPT 导出
工作台不止「问答 + 操作闭环」,还内置**内容生成**能力,把 agent 拉到的数据一键沉淀为可交付的报表与演示文稿。这是从「操作层」延伸到「汇报层」的关键补充。
### 6.5.1 报表制作
*   **自然语言生成报表**:「生成 6 月 3 号线 OEE 周报」→ agent 拉时序 + 业务数据,自动排版成含图表、指标卡、结论的完整报表。
*   **可编辑 + 回流**:报表以 Generative UI 组件呈现,用户可直接改标题、增删图表、调筛选,状态回流 agent 重算。
*   **模板化**:预置生产日报/周报、设备 OEE、停机分析、采购/库存、财务毛利等模板;支持保存自定义模板复用。
*   **多格式导出**:Excel、PDF、在线看板链接。
*   **定时推送**:可设定「每周一早 8 点自动生成上周产线周报并推送到 Chat/邮件」。
*   **GenUI 模式**:Declarative(图表/指标由 agent 输出 schema 渲染)+ Controlled(编辑控件)。
### 6.5.2 PPT 导出
*   **一键成稿**:任意报表或分析结果一键转为演示文稿,agent 自动分页、提炼标题、生成图表页与结论页。
*   **场景**:生产例会汇报、月度经营分析、异常复盘、给管理层的项目进展。
*   **风格模板**:内置企业模板(配色/logo/版式),保证对外一致性。
*   **可编辑**:生成后仍可增删页、改文案、换图,人工润色后导出 PPTX / 在线放映链接。
*   **数据联动**:PPT 中的图表绑定实时数据源,下次打开可一键刷新到最新。
> 定位补充:工作台形成「**问数据 → 做操作 → 出报表 → 生成 PPT**」的完整闭环,一条链路从一线处置贯通到管理层汇报。

## 七、系统架构
分层架构(自上而下):

| 层 | 职责与组件 |
| ---| --- |
| 前端(生成式 UI) | CopilotKit + AG-UI。左:agent 推理流;右:动态组件画布。Shared State 双向同步。 |
| Agent 编排层 | LangGraph / CrewAI / Mastra。多 agent、sub-agent、HITL 卡点、状态机。 |
| 能力层(MCP) | MCP Servers 封装各系统能力为工具:ERP / MES / 时序查询 / 工单 / 审批。 |
| 数据层 | 时序库(TDengine/InfluxDB)+ 业务库 + 向量库(RAG)+ 审计库(留痕/回溯)。 |
| 既有系统 | SAP / 用友 / MES / SCADA / WMS。 |

**关键数据流**:用户输入或告警触发 → 编排层规划步骤、调 MCP 工具拉数据 → agent 输出结构化结果并渲染为组件 → 用户在组件上编辑,Shared State 回流 → 高风险动作 HITL 确认 → MCP 回写既有系统 → 全程写入审计库,支持回溯与回滚。

## 八、Generative UI 设计规范
**三种模式的使用策略(原则:风险越高,UI 越受控):**

| 模式 | 何时用 | 风险控制 |
| ---| ---| --- |
| Controlled 受控 | 高风险:改单据、下工单、审批 | 只能填预设组件,行为可预测 |
| Declarative 声明式 | 数据展示:报表、图表、时间轴 | 输出 JSON schema,前端校验后渲染 |
| Open-ended 开放式 | 探索性:自由分析、孪生 overlay | 沙箱隔离,只读为主 |

**需预置的组件库**:诊断卡片、实时曲线图、停机时间轴、归因瀑布图、可编辑单据表单、对比合并面板、排程日历、工单卡片、审批卡片、拓扑/孪生视图。

* * *
## 九、数据与集成
**数据源**:时序数据(传感器/SCADA/IoT 网关 → TDengine/InfluxDB);业务数据(ERP 订单/库存/财务、MES 工单/排产、WMS);知识数据(SOP、设备手册、历史工单 → 向量库供 RAG)。

**集成方式**:每个既有系统封装为一个 MCP Server,暴露标准化工具(query / create / update);优先只读打通,再逐步开放写操作(写操作必须走 HITL);统一鉴权网关,MCP 工具调用继承用户权限。

* * *
## 十、安全、权限与合规
*   **权限感知**:agent 渲染与数据粒度按 RBAC 角色约束,一线与管理层看到的不同。
*   **写操作双确认**:所有回写既有系统的动作强制 HITL,高风险动作二次确认。
*   **全链路审计**:记录推理步骤、工具调用、输入输出、确认者、时间戳,不可篡改。
*   **可回滚**:关键写操作保留逆操作,支持一键回滚。
*   **数据隔离**:敏感数据脱敏后进 LLM;可选私有化/边缘部署,数据不出厂。

## 十七、平台级能力(产物生成 · 可扩展 · 可配置)
除了业务场景闭环,工作台本身要做成一个**可扩展、可配置的 Agent 平台**。这决定了它能否从「一个 demo」长成「一套底座」。

### 17.1 产物生成能力(Artifact Generation)
Agent 不只回答问题,还能直接产出可下载、可交付的文件。工业/ERP 场景高频刚需:交报表、发汇报、导数据。前面 6.5 节已定义报表与 PPT,这里补齐 Excel 与文档,并统一为一套产物能力。

| 产物 | 能力说明 | 典型场景 |
| ---| ---| --- |
| 报表制作 | 拉数据 → 生成交互式报表(图表+表格),画布内可编辑,导出 Excel/PDF/看板链接,支持定时推送 | 日报/周报、OEE 报表、生产日结 |
| PPT 制作 | 报表/分析一键成稿,自动分页提炼,套企业模板,图表绑实时数据,可编辑后导出 PPTX | 周会汇报、经营分析、评审 |
| Excel 制作 | 生成结构化表格(含公式、透视、多 sheet),导出 .xlsx,可回写业务系统 | 物料清单、排产表、对账单 |
| 文档/纪要 | 生成 SOP、工单说明、会议纪要,直接沉淀到知识库供 RAG 复用 | SOP 输出、异常复盘 |

设计要点:产物走 Generative UI 的 Declarative/Open-ended 模式,画布内预览可编辑,导出前人工微调;敏感数据导出遵循权限与脱敏规则。

### 17.2 MCP 可安装扩展(MCP Marketplace)
*   支持**动态安装 MCP Server**:ERP、MES、数据库、第三方 SaaS 的能力以 MCP 形式即插即用接入,无需改核心代码。
*   提供 MCP 市场/目录:一键安装、启用/停用、版本管理、权限授权。
*   每个 MCP 暴露的工具自动进入 agent 工具池,编排层可调用。
*   自建 MCP(内部系统)与第三方 MCP(公有能力)混用。

### 17.3 Skills 技能体系
*   把可复用业务流程封装为 **Skill**(prompt + 工具编排 + 产物模板),按需加载。
*   例:「生成生产日报」「异常根因分析」「采购单校验」都可做成 Skill,沉淀团队最佳实践。
*   Skill 支持市场化分发、版本化、按角色授权。
*   Agent 按任务意图自动加载相关 Skill,避免上下文膨胀。

### 17.4 多模型接入(Model Router)
*   支持接入多家模型:OpenAI、Anthropic、国内大模型(通义/文心/智谱等),以及**私有化/本地部署模型**(工业数据不出厂的刚需)。
*   模型路由:按任务类型/成本/时延/合规要求自动或手动选模型(高风险诊断用强模型,批量分类用轻量模型)。
