###  步骤二、后端部署

1、将epoint-web工程中，jar包中有相关js资源的都删除，重新打包替换

2、将业务相关的cssboot.js或者jsboot放置back中，后端在发送rest请求时需要此文件，当做静态文件加载 <br>

![20211026-b8834467.png](https://fdoc.epoint.com.cn/onlinedoc/rest/frame/base/attach/attachAction/getContent?isCommondto=true&attachGuid=bf6c529b-aa95-492c-b1aa-d6c01667b540)


3、将epoint-frame-sso-9.5.1-SNAPSHOT包中frame文件夹下资源全部转移至指定路径，并重新打包

4、安全模块跨域问题

- SkipReferer= 添加最终的访问IP(一般是nginx的IP),localhost,127.0.0.1

- AllowOrigin= 添加最终的访问IP(一般是nginx的IP)  <br>

###  步骤三、Nginx相关设置

#### nginx.conf中配置主要有两处，前端资源（路由至本地），后端服务（没有接入网关，配置的是后端服务的地址，如果接入网关就配置网关地址），这里建议先配置后端服务，等全部发布至网关，再进行配置网关地址

- 【未接入】网关配置示例： <br>

```
    location ~* /epoint-web/rest{
			proxy_pass http://127.0.0.1:8089;
		}
```
```
		location /epoint-web {
			alias E:/Work/Tools/epoint-web/;
		}
```

![20211027-2def9c1c.png](https://fdoc.epoint.com.cn/onlinedoc/rest/frame/base/attach/attachAction/getContent?isCommondto=true&attachGuid=06cd8c9a-6575-4fc0-903e-d95be8a2fb2f)


- 【接入】网关配置示例： <br>

```
    location ^~ /epoint-web/rest{
			proxy_pass http://127.0.0.1:8088/epoint-gateway/projectweb/;
		}
```

```
		location /epoint-web {
			alias E:/Work/Tools/epoint-web/;
		}
```

### 步骤四、访问

测试访问需要启动nginx和back，然后通过nginx的代理方式去访问。（例如上述步骤配置完成后，访问即为127.0.0.1:8888/epoint-web）。

登录未对接，登录页面需要改成.html后缀从而访问 http://localhost:8888/epoint-web/frame/pages/login/login.html，访问前台资源都变成.html页面，所以在系统启动之前先设置系统参数，及修改原生url拼接地址：

- 配置系统参数：dto_with_suffix=1

- 在拼url的地方调用 Util.addHtmlToUrl(url)，示例：url = Util.addHtmlToUrl(url);（注意，调用工具类就无需自行判断系统开启前后端分离部署模式，直接使用即可）

## maven插件的方式实现页面资源与class文件的分包方式打包

- 排除资源文件：

```xml
<plugin>
	<groupId>org.apache.maven.plugins</groupId>
	<artifactId>maven-jar-plugin</artifactId>
	<version>2.4</version>
	<configuration>
	    <excludes>
            <!-- 排除指定文件夹META-INF -->
		    <exclude>META-INF/**</exclude>
		</excludes>
	</configuration>
</plugin>
```

- 指定文件复制：


```xml
<plugin>
	<groupId>org.apache.maven.plugins</groupId>
	<artifactId>maven-resources-plugin</artifactId>
	<version>2.6</version>
	<executions>
		<execution>
			<id>copy-and-filter-allatori-config</id>
			<phase>package</phase>
			<goals>
				<goal>copy-resources</goal>
			</goals>
			<configuration>
				<outputDirectory>${basedir}/target</outputDirectory>
				<resources>
					<resource>
						<directory>${basedir}/src/main/resources</directory>
						<includes>
							<!-- 复制文件到target -->
							<include>allatori.xml</include>
							<!-- 复制文件夹到target -->
							<include>META-INF/**</include>
							<include>security/**</include>
						</includes>
						<filtering>true</filtering>
					</resource>
				</resources>
			</configuration>
		</execution>
	</executions>
</plugin>
```

## 分离部署后接入网关

### 步骤一、开启网关服务

先部署一个网关，并同时开启集群模式，相关配置见https://fdoc.epoint.com.cn/onlinedoc/rest/d/MJb6ze

### 步骤二、action发布至网关

先将后台action所指定的方法全部发布至网关（需要更新至最新的epoint-api-manage）

action发布网关地址：
http://localhost:8080/epoint-web/frame/pages/apimanage/scube/pages/dist/#/?resttype=action

（这块后期考虑网关服务初始化时自动发布，无需手动点击进行发布）

### 步骤三、 配置响应的Nginx地址，通过网关路由到相关的后台地址


```
location ^~ /epoint-web/rest{
			proxy_pass http://127.0.0.1:8088/epoint-gateway/projectweb/;
}
```
