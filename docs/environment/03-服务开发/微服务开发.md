## 服务开发与消费

### 定义服务接口
- 1、在`epoint-microservice-provider-api`工程中新建interface：`ITestMicroService`；
- 2、在`ITestMicroService`中声明服务方法`hello`；
- 3、给`ITestMicroService`中添加`RequestMapping、RestController、ApiDescription、FeignClient`注解。


```java
package com.epoint.testmicro.api;

import com.epoint.api.manage.annotation.ApiDescription;
import org.springframework.cloud.openfeign.FeignClient;
import org.springframework.web.bind.annotation.*;
import org.springframework.web.bind.annotation.PostMapping;
import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.RequestMethod;
import org.springframework.web.bind.annotation.RequestParam;

/**
 * 服务提供者接口
 */
@RestController
@RequestMapping("/testmicroservice")
@ApiDescription({"测试微服务","新点内部测试微服务"})
@FeignClient(name = "${epointmicroservice:epointmicroservice}", contextId = "testmicroservice")
public interface ITestMicroService {

    @PostMapping("/hello")
    @ApiDescription("helloworld")
    public String hello(@ApiDescription("参数")@RequestParam(value = "params", required = true) String params);

}


```

#### @FeignClient
- <font style="color:red">`name`</font>属性用来指定所属服务名称，写成kv的形式，建议kv保持一致。注意这个名字需要与`epoint-microservice-provider-web`工程下`application.properties`中的定义保持一致：`spring.application.name=`<font style="color:red">`epointmicroservice`</font>
- <font style="color:red">`contextId`</font>一般指定为当前服务接口名称即可，必须保证<font color="red">全局唯一</font>。

更多详细内容参见[[FeignClient注解说明](https://fdoc.epoint.com.cn/onlinedoc/rest/d/FZFb6r "FeignClient注解说明")](?file=015-核心框架/050-通用技术/022-rest接口/020-开发指南/008-注解说明#@FeignClient)

#### @ApiDescription
接口文档注解，开发时标注好之后，可以自动生成在线文档，详细内容参见[[接口文档开发指南](https://fdoc.epoint.com.cn/onlinedoc/rest/d/RbeURz "接口文档开发指南")](?file=010-项目介绍/020-技术规范/015-接口开发/050-接口文档规范)



### 编写服务实现
- 在`epoint-microservice-provider-service`工程中新建class ：`TestMicroService`并实现接口`ITestMicroService`
- 添加`@Component`注解

```java
package com.epoint.testmicro.rest;

import com.epoint.testmicro.api.ITestMicroService;
import org.springframework.stereotype.Component;
import com.epoint.core.utils.string.StringUtil;

/**
 *  服务接口实现
 */
@Component
public class TestMicroService implements ITestMicroService {

    @Override
    public String hello(String params) {
        if(StringUtil.isNotBlank(params)){
            return params;
        }
        else{
            return "hello world";
        }
    }

}
```


### 编写服务消费
- 在`epoint-microservice-consumer`工程下新建一个消费者类：`TestMicroServiceAction`继承`BaseController`;
- 在`TestMicroServiceAction`中分别用两种不同的调用方式`test1、test2`来实现服务调用;
- 为方便后续测试，为2个测试方法添加`@RequestSupport(type = {RequestType.GET })`注解，让其可以直接用浏览器的GET方式来请求。

```java
package com.epoint.testmicro.action;

import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.context.annotation.Scope;
import org.springframework.web.bind.annotation.RestController;

import com.epoint.basic.controller.BaseController;
import com.epoint.core.dto.annotation.RequestSupport;
import com.epoint.core.dto.annotation.RequestType;
import com.epoint.core.utils.container.ContainerFactory;
import com.epoint.testmicro.api.ITestMicroService;

@RestController("testmicroserviceaction")
@Scope("request")
public class TestMicroServiceAction extends BaseController
{
    /**
     * 注入接口服务
     */
    @Autowired
    private ITestMicroService testmicroservice;

    @Override
    public void pageLoad() {

    }

    @RequestSupport(type = {RequestType.GET })
    public String test1() {
        // 通过注入接口的方式进行服务调用
        return testmicroservice.hello("");
    }

    @RequestSupport(type = {RequestType.GET })
    public String test2() {
        // 通过获取到接口实例对象进行服务调用
        ITestMicroService testService = ContainerFactory.getContainInfo().getComponent(ITestMicroService.class);
        return testService.hello("hello world");
    }
}
```

## 服务配置
- 新点提供了3种服务互通方式：直连、基于开源注册中心（nacos）、基于自研注册中心（soa），你可以任意选择下面一种方式来进行配置。


| 服务互通方式 | 优势 | 劣势 | 备注 |
| :-----: | :-----:| :------ |:------ |
| 直连  |  简单直观，不需要额外部署注册中心。 |需要进行点对点的服务地址配置，没有自动发现路由能力，一旦服务地址变更，需要手动修改配置；没有服务负载均衡能力，负载均衡只能通过nginx代理实现。 | 推荐本地开发测试使用，或者是一些小型项目，异构系统不多的情况使用。 |
| 注册中心(nacos)  | 支持服务自动发现路由；支持负载均衡；技术主流标准，三方对接集成方便。 |需要单独部署nacos注册中心。 | 推荐在客户比较在意纯粹微服务技术，或者需要与三方厂商共同构建微服务的大型项目中进行使用。 |
| 注册中心(soa)  | 支持服务自动发现路由；支持负载均衡；不需要额外部署注册中心，直接使用新点自研的SOA服务做替代即可。 | 技术不够主流标准，属于新点自研，三方需要按照新点提供的技术规范做对接整合。 | 推荐在新点主开发的项目中使用这套方案，可以节省额外部署注册中心，交付成本低。 |

### 直连模式

- 1、打开消费者工程：`epoint-microservice-consumer-web`中的`application.propertites`配置文件，在其中配置服务发布方的ip地址：`epointmicroservice=http://localhost:8081/epoint-microservice-provider-web`


### 注册中心（nacos）

- 1、安装并启动[[注册中心](https://fdoc.epoint.com.cn/onlinedoc/rest/d/6ruYv2 "注册中心")](?file=015-核心框架/070-微服务/081-微服务部署指南/015-注册中心部署指南)
- 2、打开`epoint-microservice-provider-web`和`epoint-microservice-consumer-web`工程的`pom.xml`文件，确保引入如下工程。
  ```xml
  <dependency>
    <groupId>com.alibaba.cloud</groupId>
    <artifactId>spring-cloud-starter-alibaba-nacos-discovery</artifactId>
    <version>2.2.9.RELEASE</version>
  </dependency>
  ```
- 3、打开`epoint-microservice-provider-web`和`epoint-microservice-consumer-web`工程的`application.propertites`文件，在其中配置注册中心的ip地址：`nacos.config.server=127.0.0.1:8848`，同时确保`spring.cloud.nacos.discovery.enabled=true`j 

### 注册中心（SOA）

- 1、安装并启动SOA，这里可以直接使用前面的[[启动项目](https://fdoc.epoint.com.cn/onlinedoc/rest/d/JfIRBz "启动项目")](?file=012-快速入门/005-启动项目)作为SOA来使用，注意`epointframe.properties`文件中的属性设置：<font style="color:red">`enableSoaFlag=1`</font>。
- 2、打开`epoint-microservice-provider-web`和`epoint-microservice-consumer-web`工程的`pom.xml`文件，确保引入如下工程。
  ```xml
  <dependency>
    <groupid>com.epoint.serviceframe</groupid>
    <artifactId>epoint-cloud-epaas-loadbalancer</artifactId>
    <version>1.2.0-SNAPSHOT</version>
  </dependency>
  ```
- 3、打开`epoint-microservice-provider-web`和`epoint-microservice-consumer-web`工程的`application.propertites`文件，在其中配置SOA的ip地址：`soa-app=http://localhost:8080/epoint-web`


## 启动测试
- 1、因为默认所有的服务接口都需要进行身份鉴权才能访问，为了测试简单，我们将开发的服务接口配置成匿名。打开提供者工程：`epoint-microservice-provider-web`中的`EpointSSOClient.properties`配置文件，在其中配置`NoNeedAuthActions=testmicroservice/hello`

- 2、同样的消费端也需要配置，打开消费者工程：`epoint-microservice-consumer-web`中的`EpointSSOClient.properties`配置文件，在其中配置`NoNeedAuthActions=testmicroserviceaction`

- 3、以[[tomcat插件](https://fdoc.epoint.com.cn/onlinedoc/rest/d/JfIRBz "tomcat插件")](?file=012-快速入门/005-启动项目#启动tomcat插件)方式启动服务提供者：`epoint-microservice-provider-web`

- 4、以[[tomcat插件](https://fdoc.epoint.com.cn/onlinedoc/rest/d/JfIRBz "tomcat插件]")](?file=012-快速入门/005-启动项目#启动tomcat插件)方式启动服务消费者：`epoint-microservice-consumer-web`

- 5、浏览器分别访问`http://localhost:8082/epoint-microservice-consumer-web/rest/testmicroserviceaction/test1?isCommondto=true`和`http://localhost:8082/epoint-microservice-consumer-web/rest/testmicroserviceaction/test2?isCommondto=true`，可以看到都接口调用成功，且都返回了`hello world`。
