# 03-连接验证（DBeaver + 命令行）

## 目的

用“应用账号”验证你创建的数据库可连、可读写，为后端启动前排除数据库侧问题。

## 前置条件

- 你已创建数据库与账号权限：[02-创建数据库与账号权限](./02-创建数据库与账号权限.md)
- 你已安装 DBeaver（可选，但推荐）

## 操作步骤 A：命令行验证（必须做）

### 1. 用应用账号连接

把 `<DB_USER>`、`<DB_NAME>` 替换成你的值：

```bash
mysql -u<DB_USER> -p -h127.0.0.1 -P3306 <DB_NAME>
```

### 2. 执行最小读写验证

```sql
create table if not exists __connect_test__(
  id int primary key auto_increment,
  created_at timestamp default current_timestamp
);

insert into __connect_test__ values (null, now());
select * from __connect_test__ order by id desc limit 3;
```

### 3. 清理测试表（可选）

```sql
drop table __connect_test__;
```

## 操作步骤 B：DBeaver 验证（推荐）

### 1. 新建连接

- Database：MySQL
- Host：127.0.0.1
- Port：3306
- Database：`epoint_dev`
- Username：`epoint`
- Password：不要在项目里记录，保存到本机即可

### 2. 执行同样的 SQL 验证

在 SQL Editor 执行“最小读写验证”的 SQL，确认结果一致。

### 3. 你截图里的界面应该怎么填

你现在的截图里是：

- Server Host：`localhost`
- Port：`3306`
- Username：`root`
- Database 为空

建议你改成“应用账号”连接（后端也会用这个账号）：

- Server Host：`localhost`
- Port：`3306`
- Database：`epoint_dev`
- Username：`epoint`
- Password：输入你在第 02 步为 `epoint` 设置的密码

然后点击 `Test Connection ...`，成功后再 `Finish`。

## 预期结果

- 命令行可连接成功
- 能创建表、插入、查询
- 没有权限相关报错（例如 Access denied）

## 常见问题与排查

### 1) `Access denied for user`

优先检查：

- 账号密码是否正确
- 账号的 host 是否匹配（你是从 `127.0.0.1` 连还是从 `localhost` 连）
- 是否执行了 `flush privileges;`

如果你在 DBeaver 里报错类似：

```text
Access denied for user 'epoint'@'localhost' (using password: YES)
```

按下面顺序排查（不要把密码发到聊天里）：

1. 先用命令行验证同一套账号密码是否能登录（建议用 `localhost` 以匹配账号 host）：

```bash
mysql -uepoint -p -hlocalhost -P3306 epoint_dev
```

2. 如果命令行也登录失败，说明“密码不对”或“账号认证方式不兼容”。用 root 登录后执行下面两步修复：

先检查账号插件：

```sql
select user, host, plugin from mysql.user where user = 'epoint';
```

然后重置密码（推荐同时指定为 `mysql_native_password`，可避免部分客户端的认证兼容问题）：

```sql
alter user 'epoint'@'localhost' identified with mysql_native_password by '<NEW_PASSWORD>';
flush privileges;
```

3. 回到 DBeaver，清空并重新输入密码再测试（避免 DBeaver 仍使用旧的保存密码）。

### 2) `Unknown database`

说明 `<DB_NAME>` 写错，或数据库没创建成功。

处理：

- 用管理员账号登录后 `show databases;` 确认库名
