# 03 36周学习路线

## 路线总览

默认节奏：每天 2 小时，每周 6 天，约 12 小时。

每周固定结构：

| 模块 | 建议时间 | 做什么 |
|---|---:|---|
| 学习 | 3 小时 | 看官方文档、课程、示例 |
| 跟敲 | 3 小时 | 复现示例代码 |
| 实操 | 4 小时 | 改造成自己的小任务 |
| 复盘 | 2 小时 | 写笔记、整理问题、提交产出 |

阶段目标：

| 阶段 | 周期 | 目标产出 |
|---|---:|---|
| 基础补齐 | 第 1 到 4 周 | Python 脚本、Git 使用、环境搭建、CSV 数据处理 |
| 数据分析与 ML | 第 5 到 10 周 | 工业数据清洗、异常检测、分类/回归小项目 |
| 深度学习与时序 | 第 11 到 16 周 | 时序预测 baseline、LSTM/Transformer 对比实验 |
| 大模型与 RAG | 第 17 到 22 周 | 工业知识库问答 Demo |
| API 与部署 | 第 23 到 27 周 | FastAPI 模型服务、Docker 本地部署 |
| 工业系统集成 | 第 28 到 31 周 | 模拟 MES/SCADA/DCS 数据流 |
| 综合作品集 | 第 32 到 34 周 | 工业 AI 应用平台 Demo |
| 简历与面试 | 第 35 到 36 周 | 简历、面试题、技术汇报材料 |

## 第 1 到 4 周：基础补齐

### 第 1 周：环境和 Python 基础

学习目标：

1. 安装 Python、VS Code 或 PyCharm、Git。
2. 熟悉虚拟环境、依赖安装、脚本运行。
3. 掌握变量、列表、字典、函数、文件读写。

实操任务：

1. 创建一个 `industrial-ai-learning` 练习目录。
2. 写脚本读取一个 CSV 文件并打印前 10 行。
3. 写脚本统计某个字段的最大值、最小值、平均值。
4. 写一个函数 `detect_missing_rows(file_path)`，统计缺失值数量。

验收标准：

| 验收项 | 标准 |
|---|---|
| 环境 | 能从命令行运行 `python --version` 和 `pip --version` |
| 脚本 | 能运行 `.py` 文件并读写 CSV |
| 代码习惯 | 变量名清晰，有基础错误处理 |

### 第 2 周：Python 工程习惯

学习目标：

1. 掌握模块、包、异常处理、日志。
2. 理解配置文件和命令行参数。
3. 初步掌握 Git 提交和 README 编写。

实操任务：

1. 把第 1 周脚本拆成 `reader.py`、`cleaner.py`、`main.py`。
2. 增加日志输出，记录读取文件、清洗、保存结果。
3. 写 `README.md` 说明脚本用途和运行方式。

验收标准：

| 验收项 | 标准 |
|---|---|
| 模块化 | 不是所有代码都堆在一个文件 |
| 异常处理 | 文件不存在、字段不存在时有友好报错 |
| README | 其他人能按说明运行 |

### 第 3 周：pandas 入门

学习目标：

1. 掌握 DataFrame、Series、筛选、排序、分组。
2. 掌握缺失值、重复值、类型转换。
3. 能读取 CSV、Excel 并导出结果。

实操任务：

1. 找一份公开时序数据或自己生成模拟传感器数据。
2. 用 pandas 统计每个设备每天的平均值、最大值、报警次数。
3. 输出清洗后的 CSV 和一份简短数据质量报告。

验收标准：

| 验收项 | 标准 |
|---|---|
| 数据读取 | 能读取 CSV/Excel |
| 数据清洗 | 能处理缺失、重复、类型错误 |
| 数据报告 | 至少包含行数、字段、缺失率、异常值比例 |

### 第 4 周：NumPy 和时间序列基础

学习目标：

1. 掌握 NumPy 数组、索引、统计函数。
2. 理解时间戳、采样频率、重采样。
3. 理解工业时序中的点位、设备、传感器、报警。

实操任务：

1. 生成一份模拟 DCS 数据：时间、设备、温度、压力、流量、状态。
2. 对数据按 1 分钟、5 分钟重采样。
3. 计算滑窗均值、滑窗标准差、变化率。

验收标准：

| 验收项 | 标准 |
|---|---|
| 时序处理 | 能按时间排序、重采样 |
| 特征工程 | 能生成至少 5 个滑窗特征 |
| 阶段产出 | 完成一个可复用的数据清洗 Notebook 或脚本 |

## 第 5 到 10 周：数据分析与机器学习

### 第 5 周：数据可视化和探索分析

学习目标：

1. 掌握 Matplotlib 或 Plotly 基础。
2. 能画趋势图、分布图、箱线图、相关性热力图。
3. 能用图发现异常、漂移、周期性。

实操任务：

1. 对模拟传感器数据画趋势图。
2. 找出温度、压力、流量之间的相关性。
3. 写一份 EDA 报告。

验收标准：

| 验收项 | 标准 |
|---|---|
| 图表 | 至少 5 张图 |
| 结论 | 每张图都有一句业务解释 |
| 报告 | 能说明数据是否适合建模 |

### 第 6 周：scikit-learn 基础

学习目标：

1. 理解训练集、验证集、测试集。
2. 掌握分类、回归、指标评估。
3. 学会 Pipeline 和标准化。

实操任务：

1. 用 scikit-learn 做一个设备故障分类 baseline。
2. 尝试 Logistic Regression、Random Forest、XGBoost 或 HistGradientBoosting。
3. 输出准确率、召回率、F1、混淆矩阵。

验收标准：

| 验收项 | 标准 |
|---|---|
| 模型 | 至少训练 2 个模型 |
| 指标 | 不只看准确率，要看召回率和误报 |
| 解释 | 能说明为什么工业故障更关注漏报 |

### 第 7 周：回归和质量预测

学习目标：

1. 掌握回归模型和指标。
2. 理解 MAE、RMSE、R2。
3. 能做质量指标预测。

实操任务：

1. 用温度、压力、流量等特征预测产品质量指标。
2. 比较 Linear Regression、Random Forest Regressor、Gradient Boosting。
3. 写误差分析，找出误差最大的样本。

验收标准：

| 验收项 | 标准 |
|---|---|
| 回归模型 | 至少 3 个模型对比 |
| 指标 | MAE、RMSE、R2 齐全 |
| 分析 | 能解释模型误差可能来自哪些业务因素 |

### 第 8 周：异常检测

学习目标：

1. 理解工业异常检测场景。
2. 掌握 Isolation Forest、One-Class SVM、简单阈值法。
3. 理解误报和漏报的业务影响。

实操任务：

1. 对传感器数据注入异常点。
2. 用阈值法和 Isolation Forest 检测异常。
3. 比较两种方法的误报和漏报。

验收标准：

| 验收项 | 标准 |
|---|---|
| 异常构造 | 能模拟尖峰、漂移、长时间卡死 |
| 检测方法 | 至少 2 种 |
| 业务解释 | 能说明异常是否需要报警 |

### 第 9 周：特征工程专项

学习目标：

1. 掌握滞后特征、滑窗特征、差分特征。
2. 理解标签泄漏。
3. 能构造用于预测性维护的数据集。

实操任务：

1. 用过去 30 分钟数据预测未来 10 分钟故障风险。
2. 构造 lag、rolling、diff、ratio 等特征。
3. 检查是否用了未来数据。

验收标准：

| 验收项 | 标准 |
|---|---|
| 特征 | 至少 15 个特征 |
| 标签 | 标签定义清晰 |
| 泄漏检查 | 明确说明没有使用未来信息 |

### 第 10 周：阶段项目 1

项目：工业时序数据清洗与传统 ML baseline。

交付物：

1. 数据生成或数据来源说明。
2. 数据清洗脚本。
3. 特征工程脚本。
4. 模型训练脚本。
5. 模型评估报告。
6. 项目 README。

验收标准：

| 验收项 | 标准 |
|---|---|
| 可运行 | 按 README 能复现结果 |
| 可解释 | 能讲清每个字段和特征 |
| 可展示 | 报告里有图表、指标、结论 |

## 第 11 到 16 周：深度学习与时序预测

### 第 11 周：PyTorch 基础

学习目标：

1. 理解 Tensor、Dataset、DataLoader。
2. 掌握训练循环、loss、optimizer。
3. 能训练一个简单 MLP。

实操任务：

1. 用 PyTorch 复现一个分类模型。
2. 把工业特征数据转成 Dataset。
3. 输出训练 loss 和验证指标。

验收标准：

| 验收项 | 标准 |
|---|---|
| 训练循环 | 能独立写 forward、loss、backward、step |
| 数据加载 | 能用 Dataset/DataLoader |
| 指标 | 能保存训练日志 |

### 第 12 周：时间序列预测基础

学习目标：

1. 理解单变量和多变量时序预测。
2. 理解滑窗输入和预测步长。
3. 掌握 baseline 方法。

实操任务：

1. 用历史温度预测未来温度。
2. 实现 naive baseline、移动平均、Random Forest。
3. 用 MAE、RMSE 评估。

验收标准：

| 验收项 | 标准 |
|---|---|
| 滑窗 | 输入窗口和预测窗口定义清楚 |
| baseline | 至少 2 个 |
| 指标 | 能比较模型优劣 |

### 第 13 周：LSTM 入门

学习目标：

1. 理解 RNN/LSTM 适合处理序列。
2. 能用 PyTorch 写简单 LSTM。
3. 理解 batch、sequence length、feature dimension。

实操任务：

1. 用 LSTM 做温度或故障风险预测。
2. 调整窗口长度和隐藏层大小。
3. 比较 LSTM 与传统 baseline。

验收标准：

| 验收项 | 标准 |
|---|---|
| 维度 | 能讲清输入张量形状 |
| 对比 | 不只给 LSTM 结果，要和 baseline 比 |
| 结论 | 能说明 LSTM 是否真的更好 |

### 第 14 周：Transformer 时序入门

学习目标：

1. 理解 Attention 的基本直觉。
2. 知道 Transformer 可用于时序预测。
3. 能复现一个简化 Transformer 模型。

实操任务：

1. 用简化 Transformer Encoder 做时序预测。
2. 与 LSTM 和传统 baseline 对比。
3. 记录训练耗时和指标。

验收标准：

| 验收项 | 标准 |
|---|---|
| 模型 | 能运行简化 Transformer |
| 对比 | 有至少 3 类模型对比 |
| 反思 | 能说明深度模型不一定适合小数据 |

### 第 15 周：模型评估与解释

学习目标：

1. 理解过拟合、欠拟合、数据漂移。
2. 掌握交叉验证和时间序列切分。
3. 初步理解特征重要性和 SHAP。

实操任务：

1. 对传统 ML 模型输出特征重要性。
2. 对时序模型做误差分布分析。
3. 找出模型表现差的工况区间。

验收标准：

| 验收项 | 标准 |
|---|---|
| 评估 | 使用时间顺序切分，不随机打乱时序 |
| 解释 | 有特征重要性或误差分析 |
| 工业视角 | 能说明模型在哪些工况不可靠 |

### 第 16 周：阶段项目 2

项目：预测性维护与工业时序预测。

交付物：

1. 数据集说明。
2. 特征工程代码。
3. 传统 ML baseline。
4. LSTM 或 Transformer 模型。
5. 模型对比报告。
6. 业务汇报版 PPT 或 Markdown。

验收标准：

| 验收项 | 标准 |
|---|---|
| 完整性 | 从数据到模型到报告闭环 |
| 对比性 | 至少包含传统 ML 和深度学习 |
| 表达 | 能用业务语言说明预测提前量和风险 |

## 第 17 到 22 周：大模型与 RAG

### 第 17 周：大模型 API 和提示词

学习目标：

1. 理解 DeepSeek/Qwen 等大模型 API 调用方式。
2. 掌握 system/user prompt、结构化输出。
3. 理解温度、上下文长度、token 成本。

实操任务：

1. 写一个脚本调用大模型 API。
2. 让模型把设备报警文本分类为紧急、一般、观察。
3. 输出 JSON 格式结果。

验收标准：

| 验收项 | 标准 |
|---|---|
| API | 能完成一次真实或模拟 API 调用 |
| Prompt | 输出格式稳定 |
| 安全 | API key 不写死在代码里 |

### 第 18 周：文档清洗

学习目标：

1. 理解 RAG 的输入是高质量文档。
2. 掌握 Markdown、PDF、Word 文档清洗思路。
3. 理解 chunk 切分策略。

实操任务：

1. 准备 3 到 5 份工业规程或模拟手册。
2. 清洗成统一 Markdown。
3. 按标题和段落切分 chunk。

验收标准：

| 验收项 | 标准 |
|---|---|
| 文档 | 至少 3 份 |
| 清洗 | 去除页眉页脚、无效空行 |
| chunk | 每个 chunk 有来源 metadata |

### 第 19 周：向量化与检索

学习目标：

1. 理解 embedding 和向量检索。
2. 掌握一个本地向量库或轻量向量检索方案。
3. 理解 top_k、相似度、召回。

实操任务：

1. 把第 18 周文档向量化。
2. 输入问题，召回相关片段。
3. 输出相似度和来源。

验收标准：

| 验收项 | 标准 |
|---|---|
| 入库 | 文档能成功向量化 |
| 检索 | 查询能返回相关 chunk |
| 来源 | 返回文件名和段落编号 |

### 第 20 周：RAG 问答链

学习目标：

1. 理解检索增强生成。
2. 能把召回内容拼进 prompt。
3. 能要求模型基于资料回答。

实操任务：

1. 实现 `query -> retrieve -> generate -> answer` 流程。
2. 答案必须包含引用来源。
3. 检索不到时返回“知识库中未找到依据”。

验收标准：

| 验收项 | 标准 |
|---|---|
| 问答 | 能回答规程类问题 |
| 引用 | 答案带来源 |
| 幻觉控制 | 不知道时不乱答 |

### 第 21 周：RAG 评测与优化

学习目标：

1. 理解 RAG 质量不只看回答，还看检索。
2. 掌握基础评测集构造。
3. 能优化 chunk、top_k、prompt。

实操任务：

1. 人工写 30 个问题和标准答案来源。
2. 统计检索命中率和回答正确率。
3. 调整 chunk 大小和 top_k 对比效果。

验收标准：

| 验收项 | 标准 |
|---|---|
| 评测集 | 至少 30 问 |
| 指标 | 有检索命中和回答正确统计 |
| 优化 | 有参数调整前后对比 |

### 第 22 周：阶段项目 3

项目：工业规程 RAG 知识库。

交付物：

1. 文档清洗脚本。
2. 向量化和检索代码。
3. 问答脚本或简单界面。
4. 评测集。
5. RAG 效果报告。
6. 项目 README。

验收标准：

| 验收项 | 标准 |
|---|---|
| 可运行 | 能完成文档入库和问答 |
| 可追溯 | 答案带引用来源 |
| 可评估 | 有评测问题和失败案例 |

## 第 28 到 31 周：工业系统集成

### 第 28 周：MES/SCADA/DCS 基础
