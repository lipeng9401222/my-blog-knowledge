# 09 36周逐周详细执行计划

## 使用说明

这是一份可以直接照着执行的学习计划，适用于当前画像：

| 项目 | 默认安排 |
|---|---|
| 当前阶段 | 第 1 周开始 |
| 当前基础 | 会一点编程，但没有系统 AI/ML 工程经验 |
| 每日投入 | 2 小时 |
| 每周节奏 | 每周 6 天学习，1 天复盘或休息 |
| 目标岗位 | AI应用工程师（工业数据/工业AI方向） |
| 最终目标 | 能做工业数据清洗、预测模型、RAG知识库、FastAPI模型服务和基础部署 |

每周都按同一个闭环执行：

```text
学习输入 -> 跟敲示例 -> 改造成工业场景 -> 输出代码/文档 -> 自测考核 -> 周复盘
```

每周时间分配建议：

| 时间块 | 每周时长 | 做什么 |
|---|---:|---|
| 概念学习 | 2 到 3 小时 | 看官方文档、理解概念 |
| 代码跟敲 | 2 到 3 小时 | 跑通最小示例 |
| 独立实操 | 4 到 5 小时 | 改造成自己的任务 |
| 文档复盘 | 1 到 2 小时 | 写 README、报告、学习笔记 |

每周必须有 3 类产出：

| 产出 | 示例 |
|---|---|
| 代码产出 | `.py`、Notebook、API、Dockerfile |
| 文档产出 | README、学习笔记、模型报告、接口文档 |
| 口头表达产出 | 1 分钟项目讲解、技术复盘、面试回答 |

## 第 1 周：环境搭建与 Python 基础

### 本周定位

第 1 周不是为了“学完 Python”，而是为了建立后面 36 周都会用到的学习环境和代码习惯。你需要做到能运行 Python 脚本、读取 CSV、写简单函数、输出结果，并开始用 README 记录学习过程。

### 本周目标

| 目标 | 具体标准 |
|---|---|
| 环境可用 | 能在命令行运行 Python、pip、Git |
| 编辑器可用 | 能创建项目目录、写 `.py` 文件、运行脚本 |
| 语法入门 | 掌握变量、列表、字典、条件、循环、函数 |
| 文件处理 | 能读取 CSV 或文本文件，输出统计结果 |
| 学习习惯 | 写一份本周 README 或学习日志 |

### 本周需要学习的东西

学习目标：

record = {

### 第 3 天：条件判断和循环

学习目标：

1. 能根据阈值判断设备状态。
2. 能遍历多条数据并统计报警数量。
3. 建立“规则报警”的基本直觉。

执行任务：

1. 创建 `rule_alarm.py`。
2. 设置温度报警阈值，例如 80 摄氏度。
3. 遍历设备记录，判断是否高温。
4. 输出每台设备的报警等级。
5. 统计报警设备数量。

报警规则示例：

| 条件 | 等级 |
|---|---|
| temperature < 70 | normal |
| 70 <= temperature < 80 | notice |
| 80 <= temperature < 90 | warning |
| temperature >= 90 | critical |

考核标准：

| 考核项 | 通过标准 |
|---|---|
| if 判断 | 能正确输出不同等级 |
| for 循环 | 能处理多条记录 |
| 统计 | 能统计 warning 和 critical 数量 |
| 表达 | 能解释规则报警和模型报警的区别 |

## 第一周开始前的准备清单

如果你现在就要开始第 1 周，先确认：

1. 有一台可以安装 Python 的电脑。
