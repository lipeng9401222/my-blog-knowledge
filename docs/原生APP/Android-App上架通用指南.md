# Android App 上架及账号申请详细指南

本文档旨在提供 Android App 上架前的详细准备工作、各大主流应用市场开发者账号申请流程、HBuilderX 打包及签名指南，以及应用上架的提交流程和隐私合规要求。

## 1. 上架前置条件

在将应用提交至任何应用市场之前，必须完成以下资质的申请与备案。这些是国家合规要求的刚需。

### 1.1 软件著作权（软著）登记
软著是证明软件版权归属的重要文件，所有主流安卓应用市场均要求提供软著。

- **申请主体**：企业或个人均可申请（通常以企业名义申请更容易通过应用市场审核）。
- **申请材料**：
  - 软件源代码（前30页和后30页，每页不少于50行）。
  - 软件用户手册或设计说明书。
  - 申请人身份证明（企业营业执照副本/个人身份证复印件）。
  - 计算机软件著作权登记申请表。
- **申请周期**：普通登记通常需要 **30-60个工作日**。加急办理（需通过代理机构）最快可缩短至 **3-5个工作日**（费用较高）。
- **官方渠道**：中国版权保护中心（[https://www.ccopyright.com.cn/](https://www.ccopyright.com.cn/)）。

### 1.2 APP 备案（ICP 备案衍生）
根据工信部要求，所有在国内运行的 APP 均需要完成备案。

- **前置要求**：需要有已经备案的域名（如果 APP 后端使用该域名）。
- **备案平台**：通常通过 APP 服务器所在的云服务提供商（如阿里云、腾讯云、华为云等）的备案系统提交。
- **所需信息**：
  - APP 名称、图标。
  - APP 包名（Package Name，如 `com.company.app`）。
  - APP 签名证书的 MD5、SHA-1、SHA-256 指纹（需与打包时的 Keystore 保持一致）。
  - 域名信息及服务器 IP。
  - 主体资质信息（营业执照、法人身份证等）。
- **申请周期**：提交云服务商初审约 **1-2个工作日**，管局审核通常需要 **10-20个工作日**。

## 2. 主流安卓应用市场开发者账号申请流程

建议尽早注册各大市场的开发者账号，部分市场企业认证需要 1-3 个工作日审核。

### 2.1 华为应用市场 (Huawei AppGallery)
- **注册地址**：[华为开发者联盟](https://developer.huawei.com/)
- **认证类型**：建议企业认证。
- **所需材料**：营业执照、法人身份证正反面、对公账户信息。
- **流程**：注册账号 -> 提交企业资质 -> 华为打款验证（对公账户收到随机金额并在后台回填） -> 审核通过。
- **周期**：1-3 个工作日。

### 2.2 小米应用商店
- **注册地址**：[小米开放平台](https://dev.mi.com/)
- **认证类型**：企业认证。
- **所需材料**：营业执照、法人身份证正反面、授权书（需加盖公章）。
- **流程**：注册账号 -> 提交认证信息和授权书 -> 人工审核。
- **周期**：1-3 个工作日。

### 2.3 OPPO 软件商店
- **注册地址**：[OPPO 开放平台](https://open.oppomobile.com/)
- **认证类型**：企业认证。
- **所需材料**：营业执照原件扫描件、法人/代理人身份证、企业邮箱。
- **流程**：注册 -> 填写企业信息 -> 提交审核。
- **周期**：审核较快，通常 1-2 个工作日。

### 2.4 VIVO 应用商店
- **注册地址**：[VIVO 开放平台](https://dev.vivo.com.cn/)
- **认证类型**：企业认证。
- **所需材料**：营业执照、法人身份证、授权书。
- **流程**：注册 VIVO 账号 -> 开发者认证 -> 提交企业资质 -> 审核。
- **周期**：1-3 个工作日。

### 2.5 腾讯应用宝
- **注册地址**：[腾讯开放平台](https://open.tencent.com/)
- **认证类型**：企业认证。
- **所需材料**：营业执照、法人身份证、管理员手持身份证照片、对公账户。
- **流程**：注册 QQ/微信 -> 登录开放平台 -> 提交资质 -> 审核。
- **周期**：1-3 个工作日。

### 2.6 360 手机助手
- **注册地址**：[360 移动开放平台](https://dev.360.cn/)
- **认证类型**：企业认证。
- **所需材料**：营业执照副本、法人身份证、授权书。
- **流程**：注册账号 -> 资质认证 -> 审核。
- **周期**：1-3 个工作日。

## 3. 安卓打包签名 (Keystore) 与 HBuilderX 操作

在提交各大市场前，必须使用正式的签名证书（Keystore）对 APP 进行打包。**请务必妥善保管 Keystore 文件及密码，一旦丢失将无法更新应用。**

### 3.1 生成 Keystore 证书
需要安装 Java 环境 (JDK)，使用终端执行 `keytool` 命令：

```bash
keytool -genkey -alias your_alias_name -keyalg RSA -keysize 2048 -validity 36500 -keystore your_app_name.keystore
```
- `-alias`：证书别名，建议用英文。
- `-keyalg`：加密算法，固定使用 RSA。
- `-validity`：有效期，36500 表示 100 年。
- `-keystore`：生成的证书文件名。

*执行后会提示输入密码、姓名、组织单位等信息，请如实填写并牢记密码。*

### 3.2 HBuilderX 打包具体操作说明
1. 打开 HBuilderX，选择你的 UniApp 项目。
2. 打开 `manifest.json`，在 **App 模块配置** 和 **App 权限配置** 中确认所需权限是否勾选。
3. 在 `manifest.json` 的 **App 原生插件配置** 中（如有）确认插件配置正确。
4. 顶部菜单栏选择 **发行** -> **原生App-云打包**。
5. 在弹出的打包窗口中，选择 **Android** 平台。
6. 勾选 **使用自有证书**。
7. 填写证书信息：
   - **包名**：必须与 APP 备案及各大市场填写的包名完全一致（如 `com.company.app`）。
   - **证书别名**：生成 Keystore 时填写的 `-alias`。
   - **证书私钥密码**：生成时设置的密码。
   - **证书文件**：选择刚刚生成的 `.keystore` 文件。
8. 建议勾选 **打正式包**，点击 **打包**。
9. 等待云端打包完成后，控制台会输出下载链接或直接打开本地 `.apk` 文件所在目录。

## 4. 应用上架提交流程与隐私合规

### 4.1 隐私合规检查点（非常重要，最容易被拒）
工信部对 APP 隐私合规要求极严，应用市场也会进行自动化和人工扫描：

1. **首次启动弹窗**：APP 首次启动时，**必须**弹出一个明显的隐私政策和用户协议同意窗口。
2. **拒绝策略**：如果用户点击“不同意”，应用可以退出，但**不能**在用户未同意前收集任何设备信息（如 MAC 地址、IMEI、Android ID 等）或调用网络。
3. **隐私政策常驻入口**：APP 内部必须提供查看隐私政策和用户协议的常驻入口（通常在“设置”或“关于我们”中）。
4. **账号注销功能**：APP 必须提供明确的“注销账号”功能，且注销流程不能设置不合理的障碍。
5. **第三方 SDK 披露**：隐私政策中必须详细列出所有集成的第三方 SDK 及其收集的数据类型和用途。
6. **权限按需申请**：不能在启动时强制索要非必要的权限（如相机、位置）。必须在使用到具体功能时才动态申请，并说明用途。

### 4.2 应用上架提交流程
准备好 `.apk` 包后，前往各大开放平台后台进行提交：

1. **创建应用**：填写应用名称、包名，上传应用图标（通常需准备 512x512 和 256x256 尺寸的 PNG）。
2. **完善基础信息**：
   - 应用分类（如工具、教育、生活等）。
   - 一句话简介和详细描述。
   - 应用截图（通常需要 4-5 张高清手机截图，尺寸通常为 1080x1920）。
3. **上传 APK 包**：系统会自动解析包名、版本号和签名信息。
4. **上传资质证明**：
   - 软件著作权扫描件。
   - APP 备案截图或备案号。
   - ICP 证（若涉及经营性互联网信息服务）。
5. **填写隐私政策链接**：提供可公网访问的隐私政策 HTML 链接。
6. **提交审核**：确认无误后提交审核，审核周期通常为 **1-3 个工作日**。

### 4.3 常见被拒原因分析
- **隐私合规不达标**：未配置首次启动弹窗，或在用户同意前提前初始化了第三方 SDK 导致收集了设备指纹。
- **软著信息不符**：软著上的软件名称与提交的应用名称不一致（部分市场允许提供简称证明）。
- **功能简单或崩溃**：应用主要页面为空白，或者在审核人员的测试机上发生闪退（Crash）。
- **测试账号未提供**：如果应用需要登录才能体验完整功能，必须在提审备注中提供测试账号和密码。
- **图标或截图侵权/不符**：应用截图包含其他品牌的手机外观，或者图标涉嫌侵犯他人商标权。
