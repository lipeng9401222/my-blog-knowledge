# AI应用工程师（工业数据方向）学习文档总览

## 这套文档解决什么问题

这套文档用于分析截图中的岗位：`AI应用工程师（工业数据/工业AI方向）`。它把岗位要求拆成可学习、可练习、可验收的能力清单，并给出一条从“会一点编程”到“入职能干活”的 36 周学习路线。

默认学习画像：

| 项目 | 默认值 |
|---|---|
| 当前基础 | 会一点编程，但没有系统 AI/ML 工程经验 |
| 每日投入 | 每天 2 小时 |
| 每周投入 | 每周 6 天，约 12 小时 |
| 学习目标 | 入职后能干活，而不是只会背面试题 |
| 目标岗位 | AI应用工程师，工业数据/工业AI落地方向 |

核心判断：

| 时间 | 可达到状态 |
|---|---|
| 3 个月 | 能完成 Python 数据处理和传统 ML 小项目，开始理解工业时序数据 |
| 6 个月 | 能完成可展示的工业数据清洗、预测模型、RAG 知识库 Demo，具备投递初级岗位的材料 |
| 9 个月 | 能做出较完整的工业 AI 应用 Demo，能讲清数据流、模型、接口、部署和业务价值 |

## 当前缺口提醒

把路线执行成作品集时，最容易漏掉下面 5 件事：

| 缺口 | 为什么要补 | 最小补法 |
|---|---|---|
| 数据集来源 | 面试会追问数据是否真实、字段怎么定义 | 每个项目写清模拟数据或公开数据集来源 |
| 作品集呈现 | 只写脚本不够证明交付能力 | 每个项目至少保留 README、运行命令、截图或报告 |
| 服务部署 | 岗位要求模型能对接业务系统 | FastAPI 项目至少提供 /health、/predict、/version |
| 面试表达 | 技术做了但讲不清会被低估 | 用 STAR 模板准备 1 分钟项目讲解 |
| 周复盘 | 学习路线长，容易中途失焦 | 每周记录产出、问题和下周最重要的一件事 |

## 岗位一句话画像

这个岗位不是纯算法研究岗，也不是普通后端岗，而是一个偏落地的复合工程岗：

> 用 Python 处理工业时序和文档数据，用机器学习和大模型做预测、问答、知识库、工艺优化类应用，再用 FastAPI/Flask、Docker、MES/SCADA/DCS 对接能力把模型交付给业务系统。

## 能力优先级

| 优先级 | 能力 | 为什么重要 |
|---|---|---|
| P0 | Python 数据工程 | 工业 AI 项目首先是数据项目，清洗、特征、标注做不好，后面都不稳 |
| P0 | 机器学习与时序预测 | 岗位明确写了预测性维护、质量检测、工业时序预测 |
| P0 | 大模型应用与 RAG | 岗位明确写了 DeepSeek/Qwen、提示词工程、LangChain/LlamaIndex、知识库 |
| P0 | API 服务与部署 | 岗位要求用 FastAPI/Flask 封装模型并对接平台 |
| P1 | 工业场景理解 | 加分项很多都来自 MES、SCADA、DCS、流程行业经验 |
| P1 | 文档与沟通 | 岗位要求跨部门协作、技术文档、汇报材料 |

## 学习方法总原则

1. 每周都要有一个能看见的产出，例如脚本、Notebook、接口、报告、Demo 页面。
2. 不追求一开始就学完数学和深度学习理论，先能把数据跑通，再逐步补原理。
3. 每个知识点都要和工业场景挂钩，例如设备温度、压力、流量、电流、振动、报警、工单、规程。
4. 每 4 周做一次小作品，每 12 周做一次阶段项目。
5. 所有项目都按“问题背景、数据来源、处理流程、模型方法、评估指标、部署方式、业务价值”整理，方便之后写简历。

## 最终作品集建议

36 周结束时，至少准备以下 4 个作品：

| 作品 | 对应岗位能力 |
|---|---|
| 工业时序数据清洗管道 | DCS/SCADA 数据清洗、特征工程、数据集构建 |
| 预测性维护模型 | 传统 ML、时序预测、模型评估 |
| 工业规程 RAG 知识库 | 大模型应用、文档清洗、向量化、RAG |
| 模型服务与集成 Demo | FastAPI、Docker、REST API、工业平台对接意识 |

## 每周最小执行模板

如果时间紧张，每周只做这 4 件事：

| 动作 | 建议时间 |
|---|---:|
| 阅读官方文档或课程 | 3 小时 |
| 跟敲和改造示例 | 3 小时 |
| 完成一个小练习 | 4 小时 |
| 写学习记录和复盘 | 2 小时 |

每周复盘只回答 5 个问题：

1. 这周学了什么岗位相关能力？
2. 这周产出了什么可展示内容？
3. 哪个概念还讲不清楚？
4. 哪个代码还能优化？
5. 下周最重要的一件事是什么？
